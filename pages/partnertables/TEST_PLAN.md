# TEST_PLAN: PartnerTables (partnertables)

**URL:** `/partnertables`  
**Роль:** Партнёр  
**Авторизация:** Требуется партнёрский аккаунт  
**Версия плана:** v2.0 | Сессия S293

---

## Назначение страницы

Управление столами ресторана: создание, редактирование, генерация QR-кодов, печать QR PDF. Также показывает активные TableSession (открытые столы) и позволяет закрывать их.

---

## Happy Path сценарии

### HP-1: Просмотр столов
1. Открыть `/partnertables`
2. ✅ Список столов с их кодами, зонами, QR-иконками
3. ✅ Статус сессии: открыт/закрыт/истёк (цветовая индикация)

### HP-2: Добавить стол
1. Нажать «+ Добавить стол»
2. ✅ Диалог с полями: название, код, зона, sort_order
3. Заполнить, нажать «Сохранить»
4. ✅ Стол появляется в списке
5. ✅ Генерируется уникальный код (MAX_CODE_GENERATION_ATTEMPTS = 20)

### HP-3: Редактировать стол
1. Нажать «...» (dropdown) → «Редактировать»
2. Изменить название/зону
3. Сохранить
4. ✅ Данные обновлены

### HP-4: Удалить стол (с undo)
1. Нажать «...» → «Удалить»
2. ✅ Диалог подтверждения
3. Подтвердить
4. ✅ Стол удалён. Toast с кнопкой «Отменить» (UNDO_TIMEOUT = 5 сек)
5. Нажать «Отменить» в течение 5 сек
6. ✅ Стол восстановлён

### HP-5: Скачать QR-код одного стола
1. Нажать иконку QR на столе
2. ✅ Появляется превью QR
3. Выбрать размер (small/medium/large/xlarge)
4. Нажать «Скачать»
5. ✅ Файл PNG скачан

### HP-6: Печать QR PDF для нескольких столов
1. Выбрать несколько столов (чекбоксы)
2. Нажать «Печать QR»
3. Выбрать макет (2/3/4 колонки) и размер
4. ✅ Открывается окно печати с QR-кодами всех выбранных столов
5. ✅ Задержка 250ms перед print() для корректной отрисовки QR

### HP-7: Закрыть сессию стола
1. Нажать «Закрыть стол» (кнопка на открытой сессии)
2. ✅ Диалог подтверждения
3. Подтвердить
4. ✅ Сессия закрыта, стол переходит в статус «свободен»
5. ✅ Только роли с доступом: partner_manager, partner_staff, director, managing_director, partner_owner

### HP-8: Поиск стола
1. Ввести название или код в поле поиска
2. ✅ Список фильтруется

## Edge Cases

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| E1 | Нет столов | Пустое состояние с кнопкой «Добавить» |
| E2 | Истёкшая сессия | Визуальная индикация (isSessionExpired) |
| E3 | Undo после удаления — истёк таймаут | Удаление не отменяется, undo-кнопка пропала |
| E4 | Выбрать все столы (Ctrl+A / чекбокс «все») | Все выбраны, печать для всех |
| E5 | Печать без выбора | Кнопка «Печать» недоступна или disabled |
| E6 | Код стола совпал с существующим | Генерируется новый (до 20 попыток) |
| E7 | Зоны с overflow (слишком много) | Dropdown-меню зон с прокруткой |
| E8 | BATCH_SIZE = 5 | Обновления отправляются пачками по 5 |

## Специфические проверки

- **ORDER_STEP = 1000:** шаг sort_order при реорганизации
- **getLinkId():** нормализация поля — string, number или object `{id/value/_id}`
- **isSessionExpired():** из sessionHelpers — логика истечения по `expires_at`
- **Печать QR:** `PRINT_WINDOW_RENDER_DELAY_MS = 250` — проверить что QR виден перед print
- **UI State сохраняется в localStorage:** `partnertables_ui` ключ
