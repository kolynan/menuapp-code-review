# TEST_PLAN: OrdersList

**URL:** `/orderslist`  
**Роль:** Партнёр / администратор  
**Авторизация:** Требуется  
**Версия плана:** v2.0 | Сессия S293

---

## Назначение страницы

Список всех заказов ресторана (последние 100, от новых к старым). Поддерживает поиск, фильтрацию по статусу, типу заказа и вкладкам (все/сегодня/последние 7 дней).

---

## Happy Path сценарии

### HP-1: Просмотр списка заказов
1. Открыть `/orderslist`
2. ✅ Таблица с колонками: номер, статус, тип, дата, сумма
3. ✅ Badge цветами по статусу (new=синий, in_progress=жёлтый и т.д.)
4. ✅ Тип: Hall/Pickup/Delivery

### HP-2: Поиск заказа
1. Ввести текст в поле поиска
2. ✅ Список фильтруется в реальном времени
3. Очистить
4. ✅ Полный список

### HP-3: Фильтр по статусу
1. В селекте «Статус» выбрать «in_progress»
2. ✅ Показываются только заказы «В работе»

### HP-4: Фильтр по типу
1. В селекте «Тип» выбрать «pickup»
2. ✅ Только заказы навынос

### HP-5: Вкладки периодов
1. Переключиться между «Все» / «Сегодня» / «Последние 7 дней»
2. ✅ Заказы фильтруются по дате

### HP-6: Открыть детали заказа
1. Нажать на строку заказа
2. ✅ Переход на `/orderdetails?id=ORDER_ID`

## Edge Cases

| # | Сценарий | Ожидаемое |

## Специфические проверки

- **Сортировка:** `Order.list('-created_date', 100)` — от новых к старым
- **currentUser:** загружается для прав доступа (retry: false)
- **createPageUrl:** используется для навигации (B44 utility)
- **Фильтры:** локальная фильтрация из 100 загруженных заказов
