# TEST_PLAN: Index

**URL:** `/`  
**Роль:** Публичная (любой пользователь)  
**Авторизация:** Не требуется  
**Версия плана:** v2.0 | Сессия S293

---

## Назначение страницы

Лендинг-страница. Кнопка входа → редирект в PartnerHome. Три feature-карточки. Footer. Простейшая страница без бизнес-логики.

---

## Happy Path сценарии

### HP-1: Открыть лендинг
1. Открыть `/`
2. ✅ Страница загружается без авторизации
3. ✅ Три feature-карточки с иконками
4. ✅ Footer компонент
5. ✅ Кнопка «Войти» (или аналог)

### HP-2: Нажать «Войти»
1. Нажать кнопку входа
2. ✅ `isLoggingIn = true` (состояние загрузки)
3. ✅ `handleLogin()` → `base44.auth.redirectToLogin("/partnerhome")`
4. ✅ Редирект на страницу входа B44

## Edge Cases

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| E1 | Уже авторизован | Страница всё равно показывается (нет авто-редиректа) |
| E2 | Двойное нажатие «Войти» | `isLoggingIn=true` предотвращает повторный вызов |
| E3 | Ошибка redirectToLogin | Обработчик (catch) — предположительно нет визуального error state |

---

## Специфические проверки

- **handleLogin():** единственная бизнес-логика страницы
- **isLoggingIn state:** `false` → `true` при нажатии
- **Footer:** отдельный компонент (не inline)
- **Нет PartnerShell:** публичная страница
- **Нет useQuery:** никаких API запросов при загрузке
