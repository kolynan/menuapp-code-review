# TEST_PLAN: PartnerLoyalty

**URL:** `/partnerloyalty`  
**Роль:** Партнёр  
**Авторизация:** Требуется партнёрский аккаунт  
**Версия плана:** v2.0 | Сессия S293

---

## Назначение страницы

Настройка программы лояльности (начисление баллов за заказы и отзывы) и системы скидок. Также показывает агрегированную статистику по всем клиентам.

---

## Happy Path сценарии

### HP-1: Включить программу лояльности
1. Открыть `/partnerloyalty`
2. ✅ Форма с настройками: loyalty_enabled, % начисления, отзывы, срок жизни баллов
3. Включить чекбокс «Программа лояльности активна»
4. Задать % начисления (например, 5%)
5. Нажать «Сохранить»
6. ✅ Toast «Сохранено», настройки применены

### HP-2: Настроить бонус за отзыв
1. Включить «Начислять баллы за отзыв»
2. Указать кол-во баллов (например, 10)
3. Сохранить
4. ✅ Настройка сохранилась

### HP-3: Настроить курс списания
1. Задать `loyalty_redeem_rate` (сколько баллов = 1 единица скидки)
2. Задать `loyalty_max_redeem_percent` (макс. % скидки баллами)
3. Сохранить
4. ✅ Значения сохранены, при перезагрузке форма заполнена правильно

### HP-4: Срок жизни баллов
1. Задать `loyalty_expiry_days` (количество дней)
2. Сохранить
3. ✅ Сохранено без ошибок

### HP-5: Включить систему скидок (независимо от лояльности)
1. Включить чекбокс «Скидка активна»
2. Задать % скидки
3. Включить/выключить «Анонимным клиентам»
4. Сохранить
5. ✅ Настройки применены

### HP-6: Просмотр статистики
1. Открыть страницу с клиентами в базе
2. ✅ Блок статистики: количество клиентов, всего начислено/потрачено/сгорело баллов, кол-во отзывов
3. ✅ Числа с правильным склонением (Балл/Балла/Баллов)

## Edge Cases

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| E1 | 0 клиентов — статистика | Показывает нули, не крашится |
| E2 | loyalty_order_percent = 0 | Форма принимает 0, сохраняется |
| E3 | loyalty_expiry_days = 0 | Баллы не сгорают (или предупреждение) |
| E4 | Отключить лояльность после включения | Настройки сохраняются, клиентам баллы не начисляются |

## Специфические проверки

- **Склонение баллов:** `pluralPoints(n, t)` — проверить на значениях 1, 2, 5, 11, 21, 0
- **Defaults:** при первом открытии — значения по умолчанию из кода (5%, 10 баллов за отзыв, 100 дней, 100% макс)
- **Статистика:** данные из LoyaltyAccount + DishFeedback (два параллельных запроса)
