# TEST_PLAN: AdminPartners

**URL:** `/adminpartners`  
**Роль:** Глобальный администратор  
**Авторизация:** Email в вайтлисте (`ADMIN_EMAILS`)  
**Версия плана:** v2.0 | Сессия S290

---

## Назначение страницы

Управление всеми ресторанами-партнёрами в системе. Список всех Partner записей, возможность просматривать и переключать их plan (Free/Paid), видеть статистику.

---

## Happy Path сценарии

### HP-1: Список партнёров
1. Открыть `/adminpartners` (как admin email)
2. ✅ Список всех Partner (до 100, от новых к старым)
3. ✅ Для каждого: название, email, дата создания, план, статус

### HP-2: Переключить план (Free ↔ Paid)
1. Найти партнёра
2. Нажать Switch (переключатель плана)
3. ✅ Мутация обновляет Partner.plan
4. ✅ Toast «Сохранено»
5. ✅ Switch визуально обновился

### HP-3: Rate limit защита
1. Быстро переключить несколько партнёров
2. ✅ `isRateLimitError()` — при 429 нет retry

---

## Edge Cases

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| E1 | Нет партнёров | Пустой список с сообщением |
| E2 | Не admin → Access Denied | Карточка «Доступ запрещён», кнопка «Выйти» |
| E3 | Rate limit 429 | Toast, нет повторного запроса |
| E4 | Ошибка загрузки партнёров | AlertTriangle + retry |

---

## Специфические проверки

- **formatDate:** ru-RU локаль, формат dd.MM.yy HH:mm
- **Сортировка:** `-created_date`, limit=100
- **shouldRetry():** не retry при rate limit, до 2 попыток при других ошибках
- **rateLimitHit state:** при 429 показывается предупреждение пользователю

| Сценарий | Что подготовить |
