# TEST_PLAN: CartView (компонент корзины)

**Путь к компоненту:** `pages/CartView/` (и `pages/PublicMenu/components/CartView`)  
**Роль:** Гость ресторана  
**Контекст:** Открывается как Drawer внутри PublicMenu (`/x?...`)  
**Версия плана:** v2.0 | Сессия S293

---

## Назначение

CartView — компонент корзины гостя. Показывает блюда в корзине, историю заказов (Мои / Стол), позволяет оформить заказ, применить баллы лояльности, оставить отзывы на блюда.

---

## Состояния CartView

| Состояние | Описание |
|-----------|----------|
| **Корзина** (cart не пустая) | Блюда + кнопка «Заказать» |
| **Мои заказы** | История заказов текущего гостя |
| **Стол** | Заказы всех гостей за столом |
| **Счёт** | Итог + кнопка «Попросить счёт» |
| **Верификация стола** | Ввод кода (если hallGuestCodeEnabled) |

---

## Happy Path сценарии

### HP-1: Просмотр корзины
1. Добавить блюда в меню
2. Открыть CartView
3. ✅ Список блюд с количеством, ценой
4. ✅ Кнопки «-» и «+» для изменения количества
5. ✅ Итоговая сумма

### HP-2: Изменить количество блюда
1. Нажать «-» → количество уменьшается
2. Нажать «-» до 0 → блюдо удаляется из корзины
3. Нажать «+» → количество увеличивается
4. ✅ Итог пересчитывается

### HP-3: Оформить заказ
1. Нажать «Заказать»
2. ✅ isSubmitting = true (кнопка заблокирована)
3. ✅ Заказ отправлен, корзина очищена
4. ✅ Появляется в истории «Мои заказы»

### HP-4: Таб «Мои заказы»
1. Переключиться на вкладку «Мои»
2. ✅ История заказов текущего гостя
3. ✅ Группы: «В работе» (раскрыто), «Выдано» (свёрнуто по умолчанию)
4. Нажать на «Выдано»
5. ✅ Список раскрывается

### HP-5: Таб «Стол» (CV-14/CV-56)
1. Переключиться на вкладку «Стол»
2. ✅ Заказы всех гостей за столом (sessionOrders)
3. ✅ Отображается общая сумма по столу

### HP-6: Лояльность — применить баллы
1. В CartView ввести email
2. ✅ Загружается loyaltyAccount (earnedPoints, maxRedeemPoints)
3. Ввести количество баллов для списания
4. ✅ Скидка отображается в итоге (pointsDiscountAmount)

### HP-7: Оставить отзыв на блюдо
1. В «Мои заказы» нажать на звезду рядом с блюдом
2. ✅ ReviewDialog открывается
3. Оценить, написать комментарий, сохранить
4. ✅ Иконка блюда меняется (отзыв оставлен)

### HP-8: Верификация стола (если включена)
1. Открыть CartView без верифицированного стола
2. ✅ Показывается HallVerifyBlock с полем ввода кода
3. Ввести код → ✅ Доступ открыт

## Edge Cases

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| E1 | Корзина пустая | Таб «Мои заказы» по умолчанию, кнопка «Заказать» скрыта |
| E2 | Много блюд (10+) | Скролл работает |
| E3 | Нет других гостей — таб «Стол» | Пустое состояние или только текущий гость |
| E4 | loyaltyAccount = null | Секция лояльности скрыта (showLoyaltySection = false) |
| E5 | Двойное нажатие «Заказать» | Второй клик заблокирован (isSubmitting) |
| E6 | submitError показан → исправить и повторить | setSubmitError(null) при повторном submit |

## Специфические проверки (CV-BUG)

- **CV-BUG-05 (P0):** statusBuckets читает `stage_id`, НЕ `o.status` — «Выдано» обновляется корректно
- **CV-14/CV-56:** Таб «Стол» реализован через shadcn Tabs (не самописный)
- **CV-15:** Если 1 гость — таб «Стол» скрыт
- **expandedStatuses:** served=false по умолчанию, in_progress=true
