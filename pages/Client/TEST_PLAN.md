# TEST_PLAN: Client (ClientLogin)

**URL:** `/client`  
**Роль:** Клиент ресторана (гость)  
**Авторизация:** Не требуется  
**Версия плана:** v2.0 | Сессия S293

---

## Назначение страницы

Страница входа в личный кабинет клиента. Гость вводит email, после чего перенаправляется на `/clientaccount?email=...` для просмотра бонусов.

---

## Happy Path сценарии

### HP-1: Вход по email
1. Открыть `/client`
2. ✅ Центрированная карточка с иконкой Gift и полем email
3. Ввести корректный email (например, `user@example.com`)
4. ✅ Кнопка «Продолжить» активна
5. Нажать «Продолжить» или Enter
6. ✅ Редирект на `/clientaccount?email=user@example.com`

### HP-2: Автофокус на поле email
1. Открыть страницу
2. ✅ Поле email в фокусе автоматически (autoFocus)
3. Сразу начать вводить email

---

## Edge Cases

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| E1 | Пустое поле | Кнопка «Продолжить» задизейблена |
| E2 | Только пробелы | Кнопка задизейблена (`!email.trim()`) |
| E3 | Некорректный email | Форма HTML5 валидация (type="email") блокирует submit |
| E4 | Email без @ | HTML5 не пропустит |
| E5 | Нажать Enter в поле | Форма отправляется (form onSubmit) |

## Error States

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| R1 | Ошибка из useClientAuth | Красный текст под полем |
| R2 | isLoading = true | Spinner в кнопке, поле заблокировано |

---

## Специфические проверки

- `useClientAuth()` hook — содержит логику обработки email и submit
- При успехе hook должен выполнить navigate('/clientaccount?email=...')
- Страница не делает прямых запросов к API — всё через hook
