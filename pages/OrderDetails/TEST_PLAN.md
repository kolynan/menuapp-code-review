# TEST_PLAN: OrderDetails

**URL:** `/orderdetails?id=<order_id>`  
**Роль:** Партнёр или администратор (предположительно)  
**Авторизация:** Требуется (авторизованный аккаунт)  
**Версия плана:** v2.0 | Сессия S293

---

## Назначение страницы

Детальный просмотр конкретного заказа: все позиции, суммы, статус, тип, информация о столе, история изменений.

---

## Happy Path сценарии

### HP-1: Просмотр заказа
1. Открыть `/orderdetails?id=ORDER_ID`
2. ✅ Таблица с позициями заказа (название, кол-во, цена)
3. ✅ Итоговая сумма, тип (Hall/Pickup/Delivery), статус (badge с цветом)
4. ✅ Статус оплаты (Не оплачен/Оплачен/Частично)

### HP-2: Навигация назад
1. Нажать «← Назад» (ArrowLeft)
2. ✅ Переход на предыдущую страницу (Link to back или navigate)

## Edge Cases

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| E1 | Нет id в URL | Заказ не загружается, ошибка |
| E2 | Несуществующий id | «Заказ не найден» или AlertCircle |
| E3 | Заказ без позиций | Пустая таблица или сообщение |
| E4 | Цена null/undefined | formatPrice(null) → «0.00» |

## Error States

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| R1 | Ошибка загрузки заказа | AlertCircle + сообщение об ошибке |
| R2 | Ошибка загрузки позиций | Секция позиций с ошибкой |
| R3 | isLoading | Spinner (Loader2) |

## Специфические проверки

- **formatPrice():** `Number(value ?? 0).toFixed(2)` — 2 знака после запятой
- **format(date):** дата из date-fns в формате dd.MM.yyyy или другом
- **orders?.[0]:** первый элемент из filter — не прямой .get()
