# TEST_PLAN: PublicMenu (x.jsx)

**URL:** `/x?partner=...&mode=hall&lang=RU`  
**Роль:** Гость (клиент ресторана) — без авторизации  
**Авторизация:** Не требуется  
**Версия плана:** v2.0 | Сессия S290

---

## Назначение страницы

Публичное меню ресторана для гостей. Поддерживает три режима: зал (hall), навынос (pickup), доставка (delivery). Гости выбирают блюда, отправляют заказ, оставляют отзывы, вызывают официанта.

---

## URL-параметры

| Параметр | Обязательный | Пример |
|----------|-------------|--------|
| partner | ✅ | `69540a85f2492cff3e46a283` |
| mode | нет | `hall` / `pickup` / `delivery` |
| lang | нет | `RU` / `EN` / `KK` |
| table | нет (только hall) | `TABLE_ID` |

**Тестовый URL:** `https://menu-app-mvp-49a4f5b2.base44.app/x?partner=69540a85f2492cff3e46a283&mode=hall&lang=RU`

## Happy Path сценарии

### HP-1: Просмотр меню (Hall mode)
1. Открыть тестовый URL
2. ✅ Заголовок с логотипом ресторана
3. ✅ Категории как чипы для фильтрации
4. ✅ Блюда с фото, ценой, рейтингом
5. ✅ Переключение языка (если ресторан поддерживает несколько)

### HP-2: Добавить блюда в корзину и оформить заказ
1. Нажать «+» на блюде
2. ✅ Блюдо добавлено в корзину, кнопка StickyCartBar появилась внизу
3. Нажать на StickyCartBar «Корзина (N) → X ₸»
4. ✅ Открывается CartView (drawer)
5. Проверить содержимое корзины, нажать «Заказать»
6. ✅ Заказ отправлен, экран подтверждения

### HP-3: Верификация стола (hall mode + код)
1. Открыть меню с `?mode=hall&table=TABLE_ID` (если включён guest code)
2. ✅ Блок ввода кода стола
3. Ввести корректный код
4. ✅ Доступ к меню открыт (HallVerifyBlock пройден)

### HP-4: Режим навынос (pickup)
1. Открыть `/x?partner=...&mode=pickup`
2. ✅ Видны только блюда, доступные для навынос
3. ✅ Нет поля «стол», нет верификации
4. Оформить заказ
5. ✅ Заказ с типом pickup

### HP-5: Переключение режимов (ModeTabs)
1. Если ресторан поддерживает несколько каналов — видны вкладки
2. Переключиться между режимами
3. ✅ Список блюд обновляется (фильтрация по каналу)

### HP-6: Вызов официанта
1. Нажать кнопку HelpFab (плавающая кнопка)
2. ✅ Открывается HelpModal
3. Выбрать тип помощи
4. ✅ ServiceRequest создан, официант уведомлён

### HP-7: Оставить отзыв на блюдо
1. После оформления заказа — блюда с иконкой оценки
2. Нажать на звезду
3. ✅ ReviewDialog открывается
4. Выбрать оценку + написать комментарий
5. ✅ Отзыв сохранён

### HP-8: Программа лояльности
1. Открыть CartView, ввести email для лояльности
2. ✅ Показывает сколько баллов будет начислено
3. Применить имеющиеся баллы (если есть)
4. ✅ Скидка отображается в итоге

## Edge Cases

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| E1 | Нет блюд в меню | EmptyMenuState с сообщением |
| E2 | Неверный partner ID | ErrorState с сообщением |
| E3 | Блюдо недоступно (disabled) | Не показывается в выбранном режиме |
| E4 | Корзина пуста | StickyCartBar не отображается |
| E5 | Истёкшая сессия | Предупреждение, новая сессия создаётся |
| E6 | Несколько гостей за столом | CartView таб «Стол» показывает всех гостей |
| E7 | Отправить пустой заказ | Кнопка «Заказать» недоступна |
| E8 | Режим без таблицы (hall без table param) | Запрос стола или свободный режим |

## Специфические проверки

- **Три режима:** hall/pickup/delivery — `normalizeMode()` + `isDishEnabledForMode()`
- **Категории:** `sortCategoriesStable()` + `getVisibleCategoryIds()` — без скачков при обновлении
- **Сессия гостя:** `getOrCreateSession()` + `findGuestByDevice()` — восстановление при reload
- **Hooks:** useCurrency, useHallTable, useHelpRequests, useLoyalty, useReviews, useTableSession — все независимы
- **CartView:** открывается как Drawer (shadcn), не отдельная страница

---

## Связи с другими страницами

- CartView — встроенный компонент (drawer), не отдельный роут
- CheckoutView — встроенный компонент
- После заказа: OrderStatus показывается inline (не редирект на /orderstatus)
