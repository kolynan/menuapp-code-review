# TEST_PLAN: ClientMessages

**URL:** `/clientmessages`  
**Роль:** Клиент ресторана  
**Авторизация:** Email в localStorage (`loyalty_client_email`)  
**Версия плана:** v2.0 | Сессия S293

---

## Назначение страницы

Входящие сообщения от ресторана для клиента. Партнёр отправляет сообщения через PartnerClients, клиент читает их здесь. Сообщения помечаются прочитанными при открытии.

---

## Happy Path сценарии

### HP-1: Просмотр входящих
1. Войти через `/client`, ввести email
2. Открыть `/clientmessages`
3. ✅ Список сообщений с темой, датой, статусом (прочитано/нет)
4. ✅ Непрочитанные визуально выделены (иконка Circle)

### HP-2: Открыть/раскрыть сообщение
1. Нажать на сообщение
2. ✅ Текст сообщения разворачивается (expandedMessageId)
3. ✅ Сообщение помечается прочитанным (is_read = true) — mutation
4. ✅ Иконка меняется (прочитано)
5. Нажать повторно
6. ✅ Сообщение сворачивается

### HP-3: Несколько аккаунтов (несколько ресторанов)
1. Клиент с loyaltyAccount в 2 ресторанах
2. ✅ Сообщения из обоих аккаунтов объединены
3. ✅ Для каждого сообщения видно название ресторана (из partners)

### HP-4: Кнопка «Назад»
1. Нажать «← Назад»
2. ✅ navigate(-1)

## Error States

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| R1 | Ошибка загрузки аккаунтов | Toast «Ошибка аккаунтов» |
| R2 | Ошибка загрузки сообщений | Toast «Ошибка загрузки сообщений» |
| R3 | Ошибка загрузки партнёров | Toast «Ошибка партнёров», сообщения отображаются без названия |
| R4 | Ошибка markRead | Optimistic update откатывается или toast |

---

## Специфические проверки

- **localStorage key:** `loyalty_client_email` — именно так, не `client_email` или другое
- **Parallel requests:** сообщения загружаются параллельно для всех accountIds
- **Optimistic update:** markRead использует `onMutate` для немедленного обновления UI
- **Partners lazy load:** загружаются только когда известны partnerIds из сообщений
