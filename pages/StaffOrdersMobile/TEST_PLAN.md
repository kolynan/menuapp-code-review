# TEST_PLAN: StaffOrdersMobile

## Назначение страницы

**Пользователь:** Официант ресторана (мобильный телефон).
**Задача:** Видеть входящие заказы от гостей, принимать их, отслеживать статус.
**URL:** `/staffordersmobile`

---

## Happy Path сценарии

### HP-1: Просмотр списка заказов
1. Открыть /staffordersmobile
2. Видит список активных заказов (карточки с именем гостя, стол, время)
3. Счётчики: Мои / Чужие / Свободные — корректные цифры
4. Табы: Активные / Завершённые — переключаются

### HP-2: Принятие заказа
1. Видит новый заказ (бейдж "НОВЫЙ")
2. Нажимает на карточку → видит детали заказа (список блюд, количество, сумму)
3. Нажимает "Принято" → заказ переходит в "Мои"
4. Статус меняется с "Заказано" на "Принято"

### HP-3: Завершение заказа
1. Заказ в статусе "Принято" / "Готово"
2. Нажимает "Обслужено" → заказ переходит в "Завершённые"
3. Счётчики обновляются

### HP-4: Получение нового заказа (real-time)
1. Официант на странице заказов
2. Гость оформляет заказ (с другого устройства/вкладки)
3. Заказ появляется в списке без обновления страницы
4. Есть уведомление / бейдж / звук

## Специфические Edge Cases

### EC-1: Неверный код стола при заказе гостя
- Гость вводит неверный код стола
- Показывается ошибка "Неверный код"
- **Гость может ввести код заново** (поле не блокируется!)
- После 3 неудачных попыток → есть альтернатива (назвать код официанту)

### EC-2: Старые заказы
- Заказы за предыдущие дни НЕ должны быть в "Активные"
- Если сессия стола закрыта — заказы этого стола в "Завершённые"

### EC-3: Много заказов одновременно
- 10+ активных заказов — список скроллится, все карточки видны
- Новый заказ появляется вверху (или есть сортировка по urgency)

### EC-4: Два официанта на один заказ
- Официант А видит заказ как "Свободный"
- Официант Б принимает этот заказ
- У Официанта А заказ пропадает из "Свободные" (или обновляется статус)

### EC-5: Развёрнутый список + новый заказ
- Один из заказов развёрнут (видны детали)
- Приходит новый заказ
- Пользователю НЕ нужно скроллить чтобы увидеть новый заказ

### EC-6: Двойной префикс "Стол Стол X"
- Имя стола не должно дублировать слово "Стол"
- Проверить для столов с именами: "Стол 1", "Зал VIP", "Терраса 3"

## Специфические Error States

### ER-1: Потеря связи
- Интернет пропал → пользователь видит индикатор "Нет связи"
- Интернет вернулся → данные обновляются автоматически

### ER-2: Устаревшая страница
- Страница открыта давно (30+ мин) → данные актуальны? Есть auto-refresh?

### ER-3: Заказ исчез
- Гость отменил заказ пока официант смотрит список → корректная обработка

## Тестовые данные

Для полного тестирования нужно:
- Минимум 1 открытый стол (с кодом)
- Минимум 1 заказ от гостя (через QR/публичное меню)
- Минимум 2 гостя (чтобы проверить "Свободные" vs "Мои")
- Блюда с разными ценами (проверка сумм)
