# TEST_PLAN: OrderStatus

**URL:** `/orderstatus?token=<public_token>`  
**Роль:** Гость (pickup/delivery)  
**Авторизация:** Не требуется  
**Версия плана:** v2.0 | Сессия S293

---

## Назначение страницы

Публичная страница отслеживания статуса заказа для клиентов навынос и доставки. Автоматически обновляется каждые 10 секунд. Показывает прогресс заказа.

---

## Статусы и пайплайн

| Статус | Шаг | Цвет | Иконка |
|--------|-----|------|--------|
| new | 0 | Синий | Package |
| accepted/in_progress | 1 | Оранжевый | Clock |
| ready | 2 | Зелёный | Check |
| served/completed | terminal | Серый | Check |
| cancelled | terminal | — | XCircle |

---

## Happy Path сценарии

### HP-1: Просмотр статуса заказа
1. Открыть `/orderstatus?token=VALID_TOKEN`
2. ✅ Загружается заказ по токену
3. ✅ Прогресс-бар с шагами (new → in_progress → ready)
4. ✅ Текущий статус выделен (цвет, иконка)
5. ✅ Информация о заказе: позиции, сумма, время

### HP-2: Автообновление
1. Подождать 10 секунд (POLL_INTERVAL_MS)
2. ✅ Статус обновляется без перезагрузки страницы
3. ✅ При достижении terminal status — polling останавливается

### HP-3: Статус «Готово» (ready)
1. Официант переводит заказ в «ready» в SOM
2. ✅ Через ≤ 10 сек страница клиента показывает «Готово» (зелёный)
3. ✅ Контактный телефон ресторана виден (если есть)

### HP-4: Заказ завершён (terminal status)
1. Заказ переведён в served/completed/cancelled
2. ✅ Прогресс завершён или отменён
3. ✅ Polling останавливается (noop после terminal)

## Edge Cases

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| E1 | Нет токена в URL | Сообщение об ошибке |
| E2 | Некорректный токен (не /^[a-z0-9]{4,20}$/) | Валидация TOKEN_RE — ошибка до запроса |
| E3 | Токен не найден | «Заказ не найден» |
| E4 | Заказ старше 24 часов | ORDER_MAX_AGE_HOURS — заказ считается недействительным |
| E5 | cancelled статус | XCircle иконка, «Отменён» |
| E6 | Телефон ресторана есть | Кнопка «Позвонить» (SAFE_TEL_RE валидация) |
| E7 | Небезопасный номер | SAFE_TEL_RE отфильтровывает XSS-попытки |

## Специфические проверки

- **POLL_INTERVAL_MS = 10000:** 10 секунд между запросами
- **TOKEN_RE:** `/^[a-z0-9]{4,20}$/` — XSS-защита токена
- **SAFE_TEL_RE:** `/^tel:[+\d\s\-().]+$/` — безопасные tel: ссылки
- **STATUS_STEPS = ["new", "in_progress", "ready"]:** для pickup/delivery (не hall)
- **Шаг 1:** accepted = in_progress (getStepIndex)
