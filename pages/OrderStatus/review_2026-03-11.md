## High Priority (P1) — 2 Fixed (from Codex)

### P1-CDX-01: `closed` status not handled — renders as "new" (FIXED)
- **Line:** ~30, ~43, ~102
- **What:** `closed` is a real order status (used in OrdersList) but OrderStatus falls back to `configs.new`, shows blue "new" badge, wrong progress step, and keeps polling forever
- **Why:** Guest with a closed order sees misleading "new order" UI
- **Fix:** Added `closed` to `TERMINAL_STATUSES`, `getStatusConfig()`, and `StatusProgress` isTerminal check
- **Source:** Codex

### P1-CDX-02: Network errors shown as "Order not found" (FIXED)
- **Line:** ~362
- **What:** `orderError || !order` merged into same not-found card; network failures mislead users into thinking their link is wrong
- **Why:** User on flaky mobile connection sees "Order not found" instead of "try again"
- **Fix:** Separated into two error blocks: network error (with retry button) vs order not found
- **Source:** Codex

## Medium Priority (P2) — 2 Fixed, 3 Noted

### P2-S110-01: Refresh button touch target below 44px (FIXED)
- **Line:** ~573
- **What:** `size="sm"` Button renders ~32px, below 44px mobile minimum
- **Fix:** Added `min-h-[44px]` to className
- **Source:** CC + Codex

### P2-S110-02: itemsTotal treats line_total=0 as falsy (FIXED)
- **Line:** ~365
- **What:** `Number(item.line_total) || (fallback)` — JS `||` treats 0 as falsy
- **Fix:** Changed to `item.line_total != null ? Number(item.line_total) : (fallback)`
- **Source:** CC + Codex

### P2-CDX-04: formatPrice hardcodes ru-RU locale (NOTED)
- **Line:** ~321
- **What:** `toLocaleString("ru-RU")` — not adaptable to user locale
- **Action:** Acceptable for KZ/RU target market. Noted in BUGS.md.

### P2-CDX-05: `new` status badge vs progress step text inconsistency (NOTED)
- **What:** Badge uses `order_status.status_new` ("Принят"), progress uses `step_received`
- **Action:** i18n translation config issue, not a code bug. Noted.

### P2-CDX-06: Secondary query failures silently degrade (NOTED)
- **What:** Partner/items/contacts don't track error; failures show blank sections
- **Action:** Graceful degradation acceptable for status page. Noted.
