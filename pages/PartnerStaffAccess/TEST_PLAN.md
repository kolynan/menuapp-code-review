# TEST_PLAN: PartnerStaffAccess

**URL:** `/partnerstaffaccess`  
**Роль:** Партнёр  
**Авторизация:** Требуется партнёрский аккаунт  
**Версия плана:** v2.0 | Сессия S293

---

## Назначение страницы

Управление доступом персонала. Позволяет создавать пригласительные ссылки/email для сотрудников, назначать им роли, просматривать статус и управлять активностью.

---

## Роли персонала

| Роль | Цвет | Описание |
|------|------|----------|
| director | Золотой | Директор (доступ к кабинету, требует email) |
| managing_director | Зелёный | Управляющий директор (кабинет, требует email) |
| partner_staff | Синий | Сотрудник (официант) |
| kitchen | Оранжевый | Кухня |
| partner_manager | Фиолетовый | Менеджер |

---

## Happy Path сценарии

### HP-1: Пригласить сотрудника по ссылке
1. Нажать «+ Добавить»
2. Выбрать роль (например, partner_staff)
3. Выбрать метод: «Ссылка»
4. Нажать «Создать»
5. ✅ Генерируется ссылка вида `/staffordersmobile?token=...`
6. ✅ Кнопка «Копировать ссылку» работает

### HP-2: Пригласить директора по email
1. Нажать «+ Добавить»
2. Выбрать роль «director» или «managing_director»
3. ✅ Метод автоматически переключается на «Email» (кабинетный доступ)
4. Ввести email
5. Нажать «Создать»
6. ✅ Приглашение отправлено, запись появилась в списке со статусом «Ожидает»

### HP-3: Просмотр списка персонала
1. Открыть страницу
2. ✅ Список сотрудников с иконками ролей, статусами (активен/ожидает/выключен)
3. ✅ Фильтр по статусу: Все/Активен/Ожидает/Выключен
4. ✅ Поиск по имени/email

### HP-4: Деактивировать сотрудника
1. Нажать «...» (меню) → «Деактивировать»
2. ✅ Статус меняется на «Выключен»
3. ✅ Ссылка перестаёт работать (если проверить /acceptinvite)

### HP-5: Удалить сотрудника
1. Нажать «...» → «Удалить»
2. ✅ Диалог подтверждения
3. Подтвердить
4. ✅ Запись исчезает из списка

### HP-6: Редактировать роль
1. Нажать «...» → «Редактировать» (если есть)
2. Изменить роль
3. Сохранить
4. ✅ Роль обновлена в списке

## Edge Cases

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| E1 | Нет сотрудников | Пустое состояние с предложением добавить |
| E2 | Роль требует email — email не введён | Кнопка «Создать» недоступна |
| E3 | Некорректный email | Валидация формы |
| E4 | Фильтр «Ожидает» при нет ожидающих | «Нет записей» |
| E5 | Генерация токена | `crypto.randomUUID()` или fallback безопасный |

---

## Error States

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| R1 | Ошибка создания приглашения | Toast «Ошибка», диалог остаётся |
| R2 | Rate limit (429) | Специальная обработка — нет повторного запроса |

---

## Специфические проверки

- **CABINET_ACCESS_ROLES:** `['director', 'managing_director']` → обязателен email
- **Безопасность токена:** `crypto.randomUUID()` с fallback через `crypto.getRandomValues`
- **Ссылка персонала:** `${origin}/staffordersmobile?token=...`
- **Rate limit detection:** `isRateLimitError()` — нет retry при 429
