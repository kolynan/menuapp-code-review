# TEST_PLAN: PartnerOrderProcess

**URL:** `/partnerorderprocess`  
**Роль:** Партнёр  
**Авторизация:** Требуется партнёрский аккаунт  
**Версия плана:** v2.0 | Сессия S293

---

## Назначение страницы

Настройка процесса обработки заказов. Партнёр управляет стадиями жизненного цикла заказа: включает/выключает промежуточные стадии, меняет их названия и цвета, задаёт доступные каналы и роли для каждой стадии.

---

## Стадии заказа (системные слоты)

| Ключ | Код | По умолч. | Блокирован |
|------|-----|-----------|------------|
| new | start | ✅ активна | 🔒 да |
| accepted | middle | ❌ | нет |
| preparing | middle | ❌ | нет |
| ready | middle | ✅ активна | нет |
| served | finish | ✅ активна | 🔒 да |

---

## Happy Path сценарии

### HP-1: Просмотр стадий
1. Открыть `/partnerorderprocess`
2. ✅ Список стадий с именами, цветами, иконками
3. ✅ Системные стадии (new, served) — без возможности удалить/отключить (заблокированы)
4. ✅ Активные/неактивные стадии визуально различимы

### HP-2: Включить промежуточную стадию
1. Найти стадию «Принято» (accepted) — по умолчанию выключена
2. Включить её (toggle/checkbox)
3. Сохранить
4. ✅ Стадия появляется в процессе заказа для официантов

### HP-3: Переименовать стадию
1. Нажать Edit на стадии (не locked)
2. Изменить название
3. Сохранить
4. ✅ Новое название видно в списке
5. ✅ В StaffOrdersMobile кнопка также обновилась

### HP-4: Изменить цвет стадии
1. В диалоге редактирования выбрать цвет из 6 вариантов (blue/orange/green/red/purple/gray)
2. Сохранить
3. ✅ Цвет обновился в карточке стадии

### HP-5: Переупорядочить стадии
1. Перетащить/переместить стадии (ChevronUp/Down кнопки)
2. Сохранить
3. ✅ Порядок сохранён (sort_order обновился)

### HP-6: Настроить доступные каналы для стадии
1. Редактировать стадию
2. Включить/выключить: зал, навынос, доставка
3. Сохранить
4. ✅ Стадия видна/скрыта для нужных каналов

## Edge Cases

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| E1 | Попытка удалить locked стадию | Кнопка недоступна (заблокирована) |
| E2 | Все промежуточные стадии выключены | Только new → served |
| E3 | Включить все 3 промежуточные | Цепочка new → accepted → preparing → ready → served |
| E4 | Пустое название стадии | Валидация, не сохраняется |

## Error States

## Специфические проверки

- **TOAST_ID = 'mm1':** единый id, дубли не накапливаются
- **ORDER_STEP = 10:** шаг сортировки при изменении порядка
- **STAGE_COLORS:** 6 цветов — проверить, что все рендерятся корректно
- **allowed_roles:** «ready» стадия — только для роли kitchen
