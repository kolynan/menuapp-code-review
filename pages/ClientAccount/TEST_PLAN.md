# TEST_PLAN: ClientAccount

**URL:** `/clientaccount?email=...`  
**Роль:** Клиент ресторана  
**Авторизация:** Email из URL-параметра  
**Версия плана:** v2.0 | Сессия S293

---

## Назначение страницы

Личный кабинет клиента с балансом баллов лояльности по всем ресторанам. Показывает список аккаунтов, историю транзакций, дату истечения баллов.

---

## Happy Path сценарии

### HP-1: Просмотр баланса
1. Войти через `/client`, ввести email
2. ✅ BalanceCard: суммарный баланс по всем ресторанам
3. ✅ RestaurantList: список ресторанов с отдельными аккаунтами
4. ✅ Переключение между ресторанами (selectedAccountId)

### HP-2: История транзакций
1. Выбрать ресторан из списка
2. ✅ TransactionList загружается (useTransactions hook)
3. ✅ Список транзакций от новых к старым
4. ✅ Ближайшее истечение баллов (useExpiration hook)

### HP-3: Нет email — редирект
1. Открыть `/clientaccount` без `?email=`
2. ✅ Немедленный редирект на `/client`

## Edge Cases

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| E1 | Email не найден в базе (нет аккаунтов) | RestaurantList пустой, баланс 0 |
| E2 | Один ресторан | Автоматически выбран первый аккаунт |
| E3 | Несколько ресторанов | Переключение между ними меняет транзакции |
| E4 | Нет транзакций у аккаунта | «История пуста» (или пустой список) |
| E5 | Баллы скоро истекают | nearestExpiry показывает предупреждение |

## Специфические проверки

- **useClientAccounts(email):** загружает все LoyaltyAccount по email
- **useTransactions(selectedAccountId):** транзакции для выбранного аккаунта
- **useExpiration(transactions, expiryDays):** nearestExpiry вычисляется из `loyalty_expiry_days` партнёра
- **totalBalance:** сумма balance всех аккаунтов
- **selectedAccount авто-выбор:** первый аккаунт при загрузке (useEffect)
