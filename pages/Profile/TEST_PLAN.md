# TEST_PLAN: Profile

**URL:** `/profile`  
**Роль:** Любой авторизованный пользователь (партнёр, персонал, admin)  
**Авторизация:** Требуется любой аккаунт  
**Версия плана:** v2.0 | Сессия S293

---

## Назначение страницы

Личный профиль пользователя. Отображает email, имя, роль в системе и название ресторана (если пользователь — партнёр). Позволяет изменить отображаемое имя (full_name).

---

## Happy Path сценарии

### HP-1: Просмотр профиля — партнёр
1. Войти как партнёр
2. Открыть `/profile`
3. ✅ Email пользователя
4. ✅ Роль (badge с цветом: partner_owner = фиолетовый, partner_manager = синий и т.д.)
5. ✅ Название ресторана (загружается отдельным запросом)
6. ✅ Поле «Имя» заполнено из full_name

### HP-2: Просмотр профиля — персонал (не партнёр)
1. Войти как официант/кухня
2. Открыть `/profile`
3. ✅ Роль partner_staff/kitchen
4. ✅ Название ресторана отображается (если partner заполнен)
5. ✅ НЕт блока с ссылкой на глобального адмна (global_admin пропускает запрос Partner)

### HP-3: Изменить имя
1. Очистить поле «Имя», ввести новое
2. ✅ Кнопка «Сохранить» активна (есть изменения)
3. Нажать «Сохранить»
4. ✅ Статус: «saving» → «success» (иконка Check)
5. ✅ Toast «Профиль сохранён»
6. Перезагрузить страницу
7. ✅ Новое имя сохранилось

### HP-4: Кнопка «Сохранить» задизейблена без изменений
1. Открыть страницу (имя загружено)
2. ✅ Кнопка «Сохранить» — disabled
3. Изменить имя на то же → disabled
4. Изменить имя → enabled

### HP-5: Навигация назад
1. Нажать кнопку «← Назад» (ArrowLeft)
2. ✅ Переход на предыдущую страницу (navigate(-1))

## Edge Cases

| # | Сценарий | Ожидаемое |
|---|----------|-----------|
| E1 | Пустое имя (только пробелы) | Кнопка «Сохранить» недоступна |
| E2 | global_admin — нет partner запроса | Название ресторана не отображается (нет лишнего запроса) |
| E3 | Роль не в ROLE_BADGE_CLASSES | Badge без специального стиля (fallback) |
| E4 | Двойное нажатие «Сохранить» | saveStatus='saving' блокирует второй вызов |
| E5 | Успешное сохранение — статус возвращается в idle | Через 2000ms после успеха |

---

## Специфические проверки

- **saveStatus lifecycle:** idle → saving → success → idle (2 сек)
- **tr() функция:** безопасный wrapper — если ключ не найден, возвращает fallback
- **ROLE_BADGE_CLASSES:** admin=красный, partner_owner=фиолетовый, partner_manager=синий, partner_staff=зелёный, kitchen=оранжевый
- **global_admin проверка:** `if (userData.partner && userData.partner !== "global_admin")` — Partner.get не вызывается для глобального админа
