# Задача: Рефакторинг Drawer (корзина/заказ) — UX улучшения

## Контекст

Drawer (ДРВ) — выдвижная панель снизу в `/x` (PublicMenu), показывает текущий заказ гостя.
По результатам UX-аудита (Claude + ChatGPT, S79) выявлены проблемы. Эта задача фиксит 5 главных.

Файл с полным UX-анализом: `outputs/ChatGPT_DrawerUX_Response_S79.md`

---

## 1) Sticky header — зафиксировать шапку ДРВ

Сейчас при скролле шапка (колокольчик + «Стол — / Гость» + кнопка ✕) уезжает вверх.

**Что зафиксировать sticky:**
- Drag handle (серая полоска)
- Кнопка ✕ (закрыть drawer) — min 44×44 hit area
- Контекст: «Стол: №X» / «Вы: Гость 🖊»

**Остальной контент** (блюда, код стола, подробности, счёт) — скроллится между sticky header и sticky footer.

---

## 2) Авто-рост ДРВ по контенту

Сейчас ДРВ фиксированной высоты — контент обрезается.

**Реализовать 2 детента:**
- **Mid (по умолчанию):** ~60% экрана — основной режим
- **Full:** ~90% экрана — для длинных списков

**Авто-переход в Full при:**
- Открытии клавиатуры (focus на input кода стола)
- Раскрытии аккордеонов «Подробности» + «Счёт» одновременно
- Более 4–5 позиций в заказе

**Drag gesture:** гость может тянуть за handle вверх/вниз для переключения между детентами.

---

## 3) Вынести «Код стола» из «Подробности» в отдельный компактный блок

Сейчас «Код стола» спрятан внутри аккордеона «Подробности» вместе со скидками/бонусами.
Это разные вещи: скидки = инфо, код стола = действие.

**Новый блок (компактный, 2 строки):**
```
┌─────────────────────────────────────┐
│  🔢  Код стола     [ _ _ _ _ _ ]    │
│       или: скажите официанту 1313   │
└─────────────────────────────────────┘
```

- Позиция: между «Ваш заказ» (блюда) и «Подробности» (скидки)
- Вторая строка: мелким серым текстом, `text-xs text-gray-400`
- Блок виден всегда (не аккордеон), но компактный
- Из «Подробности» убрать код стола — оставить только скидки/бонусы

**⚠️ Баг для проверки:** сейчас 5 ячеек ввода, но код «1313» — 4 цифры. Убедиться что формат единый (или 4, или 5).

---

## 4) 2 строки на позицию блюда (длинные названия)

Сейчас название + stepper + цена в одну строку — ломается на длинных названиях.

**Новый layout:**
```
Фирменный лагман Lanzhou          56 ₸
с говядиной и овощами
  [ — ]  1  [ + ]
```

- Строка 1: название (line-clamp 2, max 2 строки) + цена справа
- Строка 2: stepper (— qty +) слева
- Если название короткое (1 строка) — layout всё равно не ломается

---

## 5) Мелкие UX-фиксы

### 5a) «Новый заказ» → «Ваш заказ»
Заменить заголовок блока. i18n ключ: `cart.title` (проверить существующий ключ, возможно уже есть).

### 5b) «Подробности» → «Выгода: -2 ₸ +2 Б»
Переименовать заголовок аккордеона чтобы было понятнее без раскрытия.

### 5c) «Счёт → Только мне: 56 ₸» — упростить
Если гость один (нет split/общего заказа) — показывать просто `Счёт: 56 ₸` без «Только мне».

---

## Порядок блоков в ДРВ (итоговый)

```
[ Sticky header: handle + ✕ + «Стол: №X / Вы: Гость» ]

  Scrollable content:
  1. Ваш заказ (позиции + stepper, 2 строки)
  2. 🔢 Код стола (компактный блок)
  3. Подробности / Выгода (аккордеон: скидки + бонусы)
  4. Счёт (аккордеон: сплит/итого)

[ Sticky footer: ИТОГО: ... ₸ + CTA «Отправить» ]
```

## Acceptance criteria

1. Шапка ДРВ (handle + ✕ + контекст стола) фиксирована при скролле
2. ДРВ растёт по контенту (mid ~60% → full ~90%), drag gesture работает
3. «Код стола» вынесен из «Подробности» в отдельный компактный блок
4. Длинные названия блюд не ломают layout (2 строки)
5. «Новый заказ» → «Ваш заказ»
6. Footer с CTA остаётся sticky
7. Существующий функционал не сломан (заказ отправляется, скидки считаются)
