### Fix 3 — CV-76 — [ISSUE FOUND: error state is dead-end]

**Verification results:**

(a) **Double-tap prevention: OK** — Line 1168: `disabled={submitPhase !== 'idle' || cart.length === 0}` disables button during 'submitting'. Additionally, onClick guard at line 1164: `if (submitPhase !== 'idle') return;`. Double-tap fully blocked.

(b) **Error path reset: MISSING** — The `submitPhase` state machine has NO transition from 'error' back to 'idle'. Current transitions:
  - idle → submitting (line 132, when isSubmitting becomes true)
  - submitting → error (line 136, when submitError present)
  - submitting → success (line 138, when no error)
  - success → idle (line 146, after 1.5s timeout)
  - error → ??? (NO transition exists)

**Impact:** When submission fails, the CTA shows "Повторить отправку" (Retry) but the button is `disabled={submitPhase !== 'idle'}` — permanently disabled. User cannot retry without closing/reopening the drawer.

**Suggested fix (OUT OF SCOPE for B2a, log for next batch):**
```jsx
// Add after line 148:
React.useEffect(() => {
  if (submitPhase === 'error') {
    const timer = setTimeout(() => setSubmitPhase('idle'), 2000);
    return () => clearTimeout(timer);
  }
}, [submitPhase]);
```

(c) **Unmount cleanup: OK** — Line 147: `return () => clearTimeout(timer);` properly cleans up success→idle timeout. No timeout exists for error path (since there's no error→idle transition — see issue above).

### Mobile-First Check (375px)

- **Оценить chip tap target:** The chip itself is small (`px-2 py-0.5`, ~16px tall). However, it sits inside a parent `<button>` row with `min-h-[44px]`. Tapping the exact chip text is the only way to enter rating mode (vs tapping the row which toggles expand/collapse). On small screens, precision tapping a ~16px chip may be difficult. **P3 suggestion:** increase chip padding to `px-3 py-1.5` for better touch target. Not in scope for B2a.
- **Footer CTA:** `min-h-[44px]`, `w-full` — no overflow risk. Safe-area: `sticky bottom-0` with `p-4` provides adequate bottom spacing on most devices (no explicit `pb-safe` but p-4 = 16px is sufficient for non-notch devices).
- **No horizontal scroll** introduced by the fix.
