**Change 3 (filteredGroups + tabCounts):** ❌ BLOCKER — `group.requests` DOES NOT EXIST

Current code (line 3797):
```javascript
const hasActiveRequest = group.type === 'table' && activeRequests.some(r => getLinkId(r.table) === group.id);
```

Prompt's replacement code:
```javascript
const hasActiveRequest = (group.requests || []).some(r => r.status !== 'resolved' && r.status !== 'cancelled');
```

**Problem:** Group objects are constructed at lines 3700-3746 with shape `{ type, id, displayName, orders }`. There is NO `requests` property. `group.requests` will always be `undefined`, so `(group.requests || [])` will always be `[]`, making `hasActiveRequest` always `false`.

This silently breaks: tables with active requests but no active orders would no longer appear in the Active tab.

**Fix needed:** The `hasActiveRequest` computation must remain unchanged from the current code. The ONLY addition should be `hasServedButNotClosed`. Corrected code:

```javascript
// filteredGroups — ONLY add hasServedButNotClosed, keep hasActiveRequest AS-IS
const filteredGroups = orderGroups.filter(group => {
  const hasActiveOrder = group.orders.some(o => {
    const config = getStatusConfig(o);
    return !config.isFinishStage && o.status !== 'cancelled';
  });
  const hasActiveRequest = group.type === 'table' && activeRequests.some(r => getLinkId(r.table) === group.id);
  // NEW (S267): served-but-not-closed — keep in Active tab until closeSession
  const hasServedButNotClosed = group.orders.some(o => {
    const config = getStatusConfig(o);
    return config.isFinishStage && o.status !== 'closed' && o.status !== 'cancelled';
  });
  return activeTab === 'active'
    ? (hasActiveOrder || hasActiveRequest || hasServedButNotClosed)
    : (!hasActiveOrder && !hasActiveRequest && !hasServedButNotClosed);
});
```

Same pattern for tabCounts — keep `hasActiveRequest` computation unchanged, only add `hasServedButNotClosed`.
