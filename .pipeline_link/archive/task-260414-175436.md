## Fix — SOM-BUG-S270-01 (P0 CRITICAL) [MUST-FIX]: «Слишком много запросов» после «Выдать все» на 4 блюдах

### Сейчас (текущее поведение)

Официант нажимает «Выдать все (4)» или «Принять все (N)» на группе столов → B44 возвращает 429 Too Many Requests → экран с сообщением «Слишком много запросов. Подождите минуту.» Батч-операция неюзабельна даже на 4 блюдах.

### Должно быть (ожидаемое поведение)

Батч-кнопки «Принять все» / «Выдать все» (для orders и для requests) отправляют мутации **последовательно**, с задержкой ~120мс между ними. Batch на 10+ блюдах завершается без 429. UI блокирует повторный клик на время всего batch (не только на время первой мутации). После завершения batch — один `invalidateQueries`, не N.

Ref: BUGS.md § SOM-BUG-S270-01; UX decision — «Выдать все должно быть надёжным на 10-30 блюдах».

### НЕ должно быть (анти-паттерны)

- NO parallel `forEach((x) => mutation.mutate(...))` anywhere in batch code paths.
- NO `Promise.all(items.map(m => mutation.mutateAsync(...)))` — тот же fan-out.
- NO удаления existing `isRateLimitError` / `shouldRetry` / `setRateLimitHit` — оставить как safety net.
- NO изменений в single-order / single-request action handlers (`handleSingleAction`, одиночный `onClick` на кнопках «Принять» / «Выдать» на одном блюде) — они уже работают правильно.

**Helper (NEW) — `runBatchSequential` near top of SOMContent / file-scope:**

Добавить helper сразу после `isRateLimitError` (line 520):

```javascript
// Sequentially run async fn for each item with delay between.
// Prevents B44 rate-limit burst on batch operations. See SOM-BUG-S270-01.
async function runBatchSequential(items, fn, { delayMs = 120 } = {}) {
  const results = [];
  for (let i = 0; i < items.length; i++) {
    try {
      results.push(await fn(items[i], i));
    } catch (err) {
      results.push({ error: err });
      if (isRateLimitError(err)) break; // stop early on 429
    }
    if (i < items.length - 1 && delayMs > 0) {
      await new Promise((r) => setTimeout(r, delayMs));
    }
  }
  return results;
}
```
