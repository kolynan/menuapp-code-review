# UX Discussion: Order Tabs in PublicMenu — How Many Tabs? What Names?

Use @discussion-moderator agent to run this CC+Codex design session.

**Context:** PublicMenu page (`menuapp-code-review/pages/PublicMenu/`). Currently when a guest places an order, they can see their orders. Arman wants to explore whether to add tabs to split orders by "mine" vs "others at the table" vs "all".

**Live app:** https://menu-app-mvp-49a4f5b2.base44.app/x?partner=69540a85f2492cff3e46a283&mode=hall&lang=RU

---

## Background

In "В зале" (hall) mode, multiple guests sit at the same table. Each guest orders independently via their own phone. Currently the order view may show all orders mixed together.

Arman's idea: add tabs so guests can distinguish their own orders from others' orders at the table.

---

## Questions to Resolve

### Q1: How many tabs? 2 or 3?

**Option A (2 tabs — recommended by Cowork):**
- "Мой заказ" — only items this guest ordered this session
- "Весь стол" — all orders from all guests at this table

**Option B (3 tabs):**
- "Мой заказ" — only this guest's items
- "Заказы других" — only items from other guests
- "Всё" — all combined

**Option C (no tabs):**
- Show all table orders in one list, but highlight "your" items visually (e.g. bold, colored border)

For each option evaluate: (1) cognitive load for non-tech user, (2) practical usefulness in real restaurant scenario, (3) implementation complexity, (4) ЛМП in restaurant apps.

### Q2: Tab naming

If 2 tabs is chosen, evaluate these name options:
- "Мой заказ" / "Весь стол"
- "Мои блюда" / "Заказы за столом"
- "Я" / "Стол"
- (suggest better options based on ЛМП)

If 3 tabs is chosen, suggest best names.

Short names preferred — tabs must fit on mobile in one row without truncation.

### Q3: Default tab

Which tab should be active by default when guest opens the order view?
- **Option A:** "Мой заказ" — guest sees their own first (most personal)
- **Option B:** "Весь стол" — guest sees full picture of what's happening
- **Option C:** Smart default — if guest has items, show "Мой заказ"; if no items yet, show "Весь стол"

### Q4: "Весь стол" usefulness — is it actually needed?

Challenge the assumption: does a guest at a restaurant actually care what their friends ordered? Or is this feature solving a non-existent problem?

Real scenario: 4 friends at table. Guest A orders Steak. Guest B orders Pasta. Does Guest A need to see Guest B's order?

Arguments for: splitting the bill, curiosity, "what did you order?"
Arguments against: complexity, privacy, nobody actually uses it

What does ЛМП say? Do apps like Wolt group orders, Toast, or similar restaurant systems offer this?
