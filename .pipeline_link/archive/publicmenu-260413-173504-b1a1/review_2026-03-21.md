# Codex Writer Findings — PublicMenu
Chain: publicmenu-260321-200456

## Findings
1. [P1] Partner lookup swallows backend failures as "not found" — In `x.jsx:1322-1345`, both partner lookup attempts catch and suppress request errors, then `x.jsx:3035-3040` renders `error.partner_not_found` for any `null` result. A transient Base44/network failure is misreported as a bad QR link, which blocks retryable recovery. FIX: keep query errors separate from real "no partner found", and render a retryable error state for failed requests instead of collapsing everything to `null`.
2. [P1] Hall StickyCartBar ignores visit lifecycle state — In `x.jsx:2101-2144`, the sticky bar mode is derived only from `cart`, `myOrders`, and `sessionOrders`, with four states total. Context in `README.md`/`BUGS.md` shows the production flow needs visit-state-driven behavior (paid, closed, reset, bill-requested, text-mode variants). The current logic can show the wrong CTA/amount/visibility for real hall sessions. FIX: derive sticky-bar state from table-session lifecycle data, then map CTA text, subtitle, visibility, and totals from that explicit state machine.
3. [P2] Translation fallback chain does not match the documented behavior — The comment at `x.jsx:321-324` says the chain is "selected lang -> EN fallback -> RU fallback -> empty string", but `makeSafeT()` at `x.jsx:523-541` only calls the current translator and then jumps straight to the hardcoded fallback map; `tr()` at `x.jsx:1229-1231` also falls back to inline Russian strings. Missing translations can therefore leak Russian copy to non-RU guests instead of an English fallback. FIX: implement the documented locale chain explicitly, or provide locale-aware fallback dictionaries and remove page-level Russian literals as universal fallbacks.
4. [P2] Number and time formatting ignore the selected in-app language — `OrderStatusScreen.formatPrice()` uses `num.toLocaleString()` at `x.jsx:982-989`, and `formatOrderTime()` uses `toLocaleTimeString(undefined, ...)` at `x.jsx:1216-1221`. The page supports runtime language switching via `useI18n()`, but these formatters stay tied to the browser default locale. FIX: derive a locale from `lang` and pass it through to all number/time formatters, including `OrderStatusScreen`.
5. [P2] Table-code confirmation sheet regresses on narrow phones for long codes — The bottom sheet renders fixed-width code boxes (`w-10`, `gap-2`) and a fixed-width input (`w-40`) at `x.jsx:3416-3442`. For 6-8 digit table codes this layout can overflow or compress badly on small devices, which is a mobile-first regression. FIX: make the code UI responsive (`w-8 sm:w-10`, tighter gaps, optional wrapping/scaling) and size the hidden input to the configured `tableCodeLength`.
6. [P2] Table-code confirmation sheet hides retry/lockout feedback — The file defines localized keys for attempts and lockout copy (`x.jsx:466`, `x.jsx:478`), but the bottom sheet only renders a raw `codeVerificationError` paragraph at `x.jsx:3450-3451`. Guests do not see remaining attempts or countdown state when verification is throttled. FIX: lift attempt-count / locked-until metadata into `x.jsx` and render localized retry info in the sheet alongside the error text.
7. [P2] StickyCartBar animation hooks are missing in the page state — `BUGS.md` tracks missing count-bump / first-item rise behavior, and `x.jsx` never records previous cart count or "became non-empty" transitions before rendering `StickyCartBar` (`x.jsx:2101-2144`, `x.jsx:3548-3569`). Without a page-level trigger, the hall sticky bar cannot reliably animate state changes. FIX: track previous cart count / first-non-empty transitions in `x.jsx` and pass explicit animation trigger props (or stable keys) into `StickyCartBar`.
8. [P3] PM-062 is not fixable in `x.jsx` alone — The page already passes `activeColor=\"#B5543A\"` into `CategoryChips` at `x.jsx:3181-3188`, and there is no remaining `indigo` class in this file. The persisted wrong color therefore lives in the imported `CategoryChips` implementation, not in `x.jsx`, which matches the context in `BUGS.md`. FIX: update the `CategoryChips` component to actually consume `activeColor`, or inline the chip renderer into `x.jsx` if a page-only patch is required.
