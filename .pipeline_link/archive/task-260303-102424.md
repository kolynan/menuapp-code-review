# UX Discussion: Dish Card Redesign — Alignment, "+" Button, Best Practices

Use @discussion-moderator agent to run this CC+Codex design session.

**Context:** This is the PublicMenu page (`menuapp-code-review/pages/PublicMenu/`). The dish cards are displayed in a 3-column grid.

**Live app for reference:** https://menu-app-mvp-49a4f5b2.base44.app/x?partner=69540a85f2492cff3e46a283&mode=hall&lang=RU

---

## Current Problems (observed in browser)

1. **Uneven text height** — dish names with long words wrap to 2-3 lines (e.g. "New York Steak" wraps to 3 lines), while short names stay on 1 line. This causes the price, rating, and "Добавить" button to appear at **different vertical positions** across cards in the same row — looks messy and unprofessional.

2. **"Добавить" button size** — the current button is medium-sized text button. On mobile, Arman noted that the app already uses a round "+" icon button for adding items. The question is: **should the same "+" pattern be applied to the menu card grid?**

3. **Photo area inconsistency** — cards with photos look visually heavy vs cards without (showing "Фото будет позже" placeholder). The photo area height may need consistency.

---

## Questions to Resolve

### Q1: Fix misaligned buttons — which approach?

The root cause is variable-height dish names. Solutions:

- **Option A:** Fixed card height + fixed name area height (e.g. 2 lines max, truncate with "…")
- **Option B:** CSS `align-items: flex-end` on each card column — button always pins to bottom of card
- **Option C:** Minimum height on name block (reserves space for 2 lines always)

Evaluate: (1) visual result, (2) readability of long names, (3) implementation complexity in Base44/React. Which is ЛМП for restaurant menu grids?

### Q2: "+" button vs "Добавить" text button

Wolt, Glovo, and other delivery apps use a small round "+" button (bottom-right of card) for quick-add without opening the item detail. The current design uses a full-width "Добавить" button.

- **Option A:** Replace "Добавить" with a round "+" button (bottom-right corner of card, ~40px)
- **Option B:** Keep "Добавить" text button but make it full-width at the bottom of the card
- **Option C:** Remove the button from card entirely — user taps the card to open detail, then adds from there

Consider: (1) tap target size on mobile, (2) visual cleanliness of the grid, (3) discoverability for non-tech users in KZ restaurants, (4) what happens when item is already in cart (show quantity counter?).

### Q3: Photo placeholder handling

Cards without photos show "Фото будет позже" in a grey area. This looks unfinished in production.

- **Option A:** Keep as-is (shows placeholder text)
- **Option B:** Show a food icon / generic dish silhouette instead of text
- **Option C:** Show a blurred gradient placeholder (common in food apps)

What does ЛМП say? What's buildable in Base44?

### Q4: Category pill tags on card (optional enhancement)

Currently there's no visual category indicator on the card itself (category is only in the filter pills at top).

Should cards show a small category badge (e.g. "Блюдо дня" badge on featured items)?
- **Option A:** Yes, for special categories only (Блюдо дня, Популярное)
- **Option B:** No — it clutters the card
