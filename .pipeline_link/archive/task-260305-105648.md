### Фаза 1: Свёрнутая карточка (переделка)

**Hall (OrderGroupCard):**
```
┌──────────────────────────────────────────────┐
│ Стол 12                               8 мин  │
│ 🍽️ Зал  •  Новый                         ★   │
│ Стейк×1, Карбонара×1 +2                     │
│ [если есть запросы]  🔔 Счёт(1)  ✋ Официант(1)│
```

**Pickup:**

**Delivery:**
```
┌──────────────────────────────────────────────┐

│ Стейк×1, Суп×2 +3                            │
└──────────────────────────────────────────────┘
```

**Правила свёрнутой карточки:**
1. Строка 1: идентификатор (стол / номер заказа) + ⏱ возраст (мин)
2. Строка 2: иконка канала + статус (человекочитаемый!) + ★ (если favorite) + контакты (pickup/delivery)
3. Строка 3: превью блюд — первые 3 элемента `Название×Q`, если больше → `+N`
4. Строка 4 (только Hall): бейджи запросов (🔔 Счёт, ✋ Официант) — только если есть open requests
5. Иконки каналов: 🍽️ Зал / 📦 Самовывоз / 🚚 Доставка
6. **Запрещено:** системные ID (`CB-d2d`), UUID, сырые i18n ключи (`orderprocess.default.*`)
7. Overdue (>10 мин без смены статуса): ⏱ акцентом + опционально тонкая полоска слева
8. needsAction (есть actionable заказ или open requests): маркер ● рядом со статусом

### Фаза 2: Развёрнутая карточка (expand/collapse)

**Поведение:**
- Тап по свёрнутой → раскрыть
- Тап по шапке раскрытой → свернуть
- Тап по другой карточке → раскрыть новую, свернуть предыдущую (max 1 раскрыта)
- Скролл НЕ сворачивает (MVP)
- Анимация: высота 160-200ms ease-out, контент opacity 80-120ms
- При авто-обновлении данных: если раскрытая карточка ещё существует → оставлять раскрытой

**Блоки развёрнутой карточки (Hall) — порядок A→B→C→E→F→D:**

**Блок A — Активные заказы:**
```
▸ Гость 2 • 12:34              Новый (!)
  Стейк×1, Суп×2                 ЗАЛ-021
▸ Айжан  • 12:10              Готовится
  Карбонара×1 +1                  ЗАЛ-020
```
- Список активных заказов стола
- Каждый заказ: имя гостя (или "Гость N") + время + статус + блюда
- **Номер заказа (ЗАЛ-021) — мелким серым текстом справа внизу** (secondary, не главный!)
- `guestDisplayName`: `order.guest.name` → fallback `"Гость " + guest_number` → fallback `"Гость"`
- Тап по заказу → делает его selectedOrder для кнопки (Блок B)
- По умолчанию selectedOrder = самый срочный: new > ready > cooking > прочие

**Блок B — Следующее действие:**
- Одна главная кнопка для selectedOrder
- Тексты: `new` → "Принять", `cooking` → "Готово", `ready` → "Подано" (Hall) / "Выдать" (Pickup/Delivery)
- Под кнопкой: "статус A → статус B"
- Disabled если `isUpdating` → "Сохраняем…"
- Если нет `next` → кнопку скрыть

**Блок C — Запросы гостей (только Hall):**
- Открытые запросы: тип + возраст + кнопка [Закрыть]
- Если нет запросов → блок скрыт

**Блок E — Счёт стола (только Hall):**
- E0 (компактный, по умолчанию): `💰 Счёт: Гость 1 — 1200₸ · Гость 2 — 800₸ · ИТОГО: 2000₸`
- E1 (по тапу): развернуть с деталями (выполнено X + в работе Y)
- Данные: использовать `billsByGuest` и `tableTotal` из useTableSession (уже есть в коде!)
- Smart auto-expand: если есть open request с типом "Счёт" → блок E раскрыт и подсвечен
- 1 гость → только ИТОГО
- 5+ гостей → первые 3 + "Ещё N"
- 0 заказов → блок скрыт

**Блок F — Выполненные заказы (только Hall):**
- F0 (компактный): `✅ Выполнено: 4 заказа · 2 800 ₸ [Показать]`
- F1 (по тапу): последние 3 заказа + "Ещё N"
- 0 выполненных → блок скрыт

**Блок D — Действия со столом (только Hall):**
- Кнопка "Закрыть стол"
- Enabled: когда нет заказов с `internal_code` in ['new', 'cooking']
- Disabled причины: "Есть новый заказ (не принят)" / "Есть блюда в работе"
- Confirm: "Закрыть стол 12?" + описание + Отмена/Закрыть

**Блоки для Pickup/Delivery (порядок A→B):**
- Блок A: полный состав заказа (все items)
- Блок B: кнопка действия
- Контакты: имя + телефон (тап → вызов) + адрес (только delivery)

## Edge cases
- Пустой список → "Нет активных заказов" + "Новые заказы появятся автоматически" + кнопка "Обновить"
- Заказ без имени → "Гость N" (hall) / "Клиент" (pickup/delivery)
- 10+ позиций → первые 6 + "Показать ещё (N)"
- Несколько заказов на столе → список в Блоке A, selectedOrder = самый срочный
- Ошибка загрузки → error state + "Повторить"

## Acceptance criteria
1. В списке нет системных id и i18n ключей
2. Каждая карточка показывает: канал, статус, возраст, 2-3 блюда
3. Обработка заказа без перехода на отдельную страницу
4. Expand/collapse: max 1 раскрыта, анимация
5. Pickup/Delivery показывают имя/телефон/адрес
6. Hall показывает счёт по гостям и выполненные заказы
7. Номер заказа (ЗАЛ-021) — мелкий серый текст, не главный идентификатор
8. Кнопки действий с человекочитаемыми текстами
