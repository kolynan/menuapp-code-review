# Task: Add .gitattributes + check commit status

## Context
1. The repo has NO `.gitattributes` file. This causes CRLF/LF mismatch on Windows — `git status` shows 150+ false changes.
2. Commit `20c94c7` was supposedly created in S98 (66 i18n keys fix). Need to check if it exists on remote or was lost to worktree cleanup.

## Steps

### Step 1: Check if commit 20c94c7 exists
```bash
git fetch origin
git log --all --oneline | grep "20c94c7" || echo "COMMIT NOT FOUND"
```
Report the result.

### Step 2: Create .gitattributes
Create `.gitattributes` in repo root with this exact content:
```
# Auto-detect text files and normalize line endings
* text=auto

# Force LF for code files
*.jsx text eol=lf
*.js text eol=lf
*.md text eol=lf
*.txt text eol=lf
*.json text eol=lf
*.sh text eol=lf
*.css text eol=lf
*.html text eol=lf

# Binary files
*.png binary
*.jpg binary
*.gif binary
*.ico binary
```

### Step 3: Normalize existing files
```bash
git add .gitattributes
git rm --cached -r .
git reset HEAD
git add .gitattributes
```

### Step 4: Commit and push
```bash
git add .gitattributes
git commit -m "chore: add .gitattributes to fix CRLF/LF line-ending issues (KB-003)"
git push
```

IMPORTANT: Only commit `.gitattributes` itself. Do NOT `git add .` or `git add -A`. The 150+ CRLF-modified files should NOT be committed.

## Expected result
- `.gitattributes` exists in repo root
- Future file edits will auto-normalize line endings
- Report whether commit 20c94c7 was found
