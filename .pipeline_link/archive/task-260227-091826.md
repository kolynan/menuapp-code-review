File: `pages/acceptinvite/acceptinvite.jsx` (367 lines) — Accept staff invitation page.

**What this page does:**
A staff member receives a link with ?token=... via email. This page:
1. Looks up StaffAccessLink by token
2. Checks if link is active
3. If no email required → redirects to staff page (token-only access)
4. If email required → checks auth → verifies email match → accepts invite
5. Updates StaffAccessLink (invited_user, invite_accepted_at)
6. Calls base44.auth.updateMe() to set user_role and partner
7. Redirects to /partnerhome (cabinet roles) or /staffordersmobile?token= (staff roles)

**Key business logic (LOCKED DM-027):**
- Role mapping: staff_role → user_role (e.g. 'director' → 'partner_owner')
- Cabinet access: only 'director' and 'managing_director' get /partnerhome
- Security: no-email links with cabinet roles are rejected (P0 SEC rule)
- Retry: updateMe uses retryAsync(fn, 2, 500) for reliability

**Entities:** StaffAccessLink (filter by token, update accepted_at + invited_user)
**Auth:** base44.auth.isAuthenticated(), base44.auth.me(), base44.auth.updateMe(), base44.auth.redirectToLogin(), base44.auth.logout()
**i18n:** Already uses useI18n() + t() throughout

The page is in production and working. We want to find:
