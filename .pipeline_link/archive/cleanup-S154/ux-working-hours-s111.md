# UX Discussion: Working Hours Display on Partner Settings

## Context
Partner Settings page will include restaurant working hours (merged from Contacts).
Question: should working hours be shown as a simple text field or as a structured day-by-day picker?

## Options
1. **Option A: Single text field** — e.g. "Mon-Fri 9:00-22:00, Sat-Sun 10:00-23:00"
2. **Option B: Structured picker** — each day of week has open/close time inputs + toggle for closed days

## Evaluation criteria
- Mobile UX (partner uses phone)
- Ease of data entry
- Data consistency for client-facing display
- i18n (ru/en/kk)
