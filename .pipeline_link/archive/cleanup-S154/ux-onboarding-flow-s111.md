# UX Discussion: Partner Onboarding — First Steps After Registration

## Context
When a new restaurant partner registers in MenuApp, they land on PartnerHome. Currently there is no guided onboarding — the partner sees an empty dashboard and has to figure out what to do next.

## Question
What should the first-time onboarding experience look like? How do we guide a new partner from "just registered" to "menu published and ready for customers"?

## Options to discuss
1. **Checklist widget** on PartnerHome — shows steps: add menu items, set working hours, add tables, publish
2. **Step-by-step wizard** — full-screen guided flow that walks through setup
3. **Empty state prompts** — each section (menu, tables, etc.) has smart empty states with "Get started" CTAs
4. **Combination** — wizard for critical setup + checklist for remaining items

## Evaluation criteria
- Time to first published menu (minimize)
- Cognitive load for non-technical restaurant owner
- Mobile-first (partner uses phone)
- Skip-ability (some partners want to explore freely)
- i18n (ru/en/kk)
