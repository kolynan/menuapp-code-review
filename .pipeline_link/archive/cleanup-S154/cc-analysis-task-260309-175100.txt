== СОГЛАСОВАННЫЕ ПУНКТЫ ==

1. MERGE правильный — "один экран для всего ресторана"
   Телефон, адрес, email, соцсети, логотип, язык, валюта, часы — всё это "базовая
   информация о ресторане". Разделять на два места = усложнять без причины.

2. Дублирование часов работы УСТРАНЕНО
   Единственная реальная проблема текущего разделения — часы работы были в обоих местах.
   После слияния — один источник правды.

3. 8 вкладок — нормально для Settings
   Horizontal scroll tabs (SectionNav) уже работают для 7 вкладок. Добавить 8-ю —
   не проблема для мобильного. Лучше чем отдельный пункт в hamburger меню.

4. Доступность решается через редирект
   /partnercontacts уже перенаправляет на /partnersettings. Если нужно — можно
   оставить пункт "Контакты" в меню, он будет открывать Settings на нужной вкладке.

5. Нет причин держать отдельную страницу
   Отдельная страница оправдана только если:
   - контакты обновляются намного чаще остальных настроек
   - разные роли персонала имеют доступ только к контактам
   - служба поддержки отправляет владельцев напрямую на контакты
   Ни одно из этих условий в MenuApp не выполняется.
