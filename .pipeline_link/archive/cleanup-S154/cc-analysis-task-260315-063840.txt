## CC Analysis: task-260315-063840 — Fix null exit_code in parallel-write supervisor (KB-039)

### Problem
PS 5.1 returns $null for Process.ExitCode and $LASTEXITCODE on normal process completion.
The parallel-write supervisor set worker state to 'failed' when ExitCode was $null (since
$null -eq 0 is $false in PS 5.1), causing the reconciler guard on line 2053 to throw even
though both writers completed successfully with commits.

### Root Cause
Line 2042/2045: `$writerProc.ExitCode -eq 0` evaluates to $false when ExitCode is $null.
This sets state='failed', and line 2053 checks state != 'completed' -> throws.

### Fix Applied
1. Added `Get-V7NormalizedExitCode` helper to V7.Common.ps1 — converts $null to 0 via `-as [int]`
2. Applied to 15 exit code comparisons across 5 files:
