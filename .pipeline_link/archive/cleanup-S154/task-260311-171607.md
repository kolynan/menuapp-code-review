# UX Discussion: How Should Menu Categories Be Organized?

## Context
MenuManage page lets partners create and organize their restaurant menu. Currently categories are a flat list. Partners with large menus (50+ items) struggle to organize them.

## Question
How should menu categories be structured to help partners with large menus?

## Options
1. **Flat list with drag-and-drop reordering** — current approach, simple but doesn't scale
2. **Nested categories** — e.g. "Drinks > Hot Drinks > Coffee" — powerful but complex UI
3. **Tags + smart groups** — items can have multiple tags, auto-grouped by tag
4. **Sections + categories** — two-level hierarchy only (Section = "Food/Drinks", Category = "Appetizers/Main/Dessert")

## Evaluation criteria
- Scalability (works for 10 items AND 100+ items)
- Partner UX (non-technical restaurant owner on mobile)
- Client UX (customer browsing QR menu)
- i18n support (ru/en/kk category names)
