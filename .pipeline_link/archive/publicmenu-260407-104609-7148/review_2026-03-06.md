## FIX-1: BUG-S81-07 -- Duplicate restaurant name (line 3103)

**Code:**
```jsx
<PublicMenuHeader
  partner={showLogo ? { ...partner, name: undefined } : partner}
```

### Verdict: CORRECT with one minor issue

When showLogo is true, the logo+name block at lines 3119-3131 renders the name. Setting partner.name to undefined in the spread suppresses it in PublicMenuHeader. When showLogo is false, the full partner is passed so PublicMenuHeader can display the name.

**Issues found:**

1. **[P2] Shallow copy creates new object every render** (line 3103)
   - { ...partner, name: undefined } creates a new object reference on every render.
   - If PublicMenuHeader uses React.memo, this defeats memoization.
   - Suggested: memoize with useMemo.

2. **[P3] Relies on PublicMenuHeader treating undefined name gracefully** (line 3103)
   - Setting name: undefined is fine if PublicMenuHeader does partner?.name check. Standard React pattern, low risk.

**No correctness bugs found in FIX-1.**

## FIX-2: BUG-S81-03 -- CTA button not visible (lines 3320, 3433)

**Code (both drawers):**
```jsx
<DrawerContent className="flex flex-col" style={{ maxHeight: '90vh' }}>
  <div className="...shrink-0">drag handle</div>
  <div className="flex-1 overflow-y-auto min-h-0">
    <CartView ... /> or <PickupDeliveryCheckoutContent ... />
  </div>
</DrawerContent>
```

### Verdict: CORRECT -- sticky CTA works as intended

3. **sticky bottom-0 inside overflow-y-auto -- VERIFIED CORRECT**
   - CartView CTA at CartView.jsx line 1203 uses sticky bottom-0.
   - Inside overflow-y-auto, sticky elements stick to the scroll viewport bottom.
   - This is exactly the desired behavior: CTA stays pinned as user scrolls cart.

4. **[P2] Double overflow-y-auto in checkout drawer** (lines 1036 + 3433)
   - PickupDeliveryCheckoutContent has its own internal overflow-y-auto (line 1036) plus flex-col h-full overflow-hidden on its root (line 1022).
   - The outer wrapper at line 3433 also has overflow-y-auto.
   - Creates nested scroll. In practice, since the inner component manages its own height with h-full, the outer scroll should have nothing to scroll. Not actively broken, but redundant.
   - Risk: On some browsers, h-full inside a flex child might not resolve correctly. Needs real device testing.

## FIX-3: BUG-S81-01 -- Swipe-to-close (lines 3308-3318, 3421-3428)

**Code (cart drawer, line 3314):**
```jsx
if (delta > 80) setDrawerMode(null);
```

**Code (checkout drawer, line 3427):**
```jsx
if (delta > 80 && !isSubmitting) setDrawerMode(null);
```

### Verdict: BUG FOUND -- missing isSubmitting guard on cart drawer

5. **[P1] MISSING isSubmitting guard on cart drawer swipe-close** (line 3314)
   - The checkout drawer correctly checks !isSubmitting at line 3427.
   - The cart drawer at line 3314 does NOT: if (delta > 80) setDrawerMode(null)
   - If user swipes down while order is submitting, drawer closes mid-submission.
   - **Fix:** Change to: if (delta > 80 && !isSubmitting) setDrawerMode(null);

6. **[P1] Cart drawer onOpenChange also lacks isSubmitting guard** (line 3293)
   - Line 3293: onOpenChange={(open) => !open && setDrawerMode(null)}
   - Compare checkout drawer line 3399: if (!open && !isSubmitting) { ... }
   - Cart drawer can be closed via backdrop tap during submission.
   - Pre-existing bug, now more visible alongside FIX-3.
   - **Fix:** onOpenChange={(open) => { if (!open && !isSubmitting) setDrawerMode(null); }}

7. **[P2] 80px threshold -- ACCEPTABLE for v1**
   - 80px is within standard range (50-100px). Prevents accidental closes.
   - Lacks velocity consideration (fast 60px swipe vs slow 90px drag).
   - No visual feedback during drag. Both are P3 enhancements for later.

8. **[P3] touch-none on drag handle -- CORRECT**
   - Prevents browser from stealing touch events for native scroll/pan.
   - React onTouchStart/onTouchEnd fire reliably with this setting.

9. **[P3] Shared drawerDragStartY ref between two drawers** (line 1233)
   - Both drawers use the same useRef(0). Since only one is open at a time (drawerMode is either cart or checkout), no conflict possible.

10. **[P3] No null check on changedTouches[0]** (lines 3313, 3426)
    - In rare edge cases (touch cancelled by OS), changedTouches might be empty.
    - Negligible risk on modern mobile browsers.
