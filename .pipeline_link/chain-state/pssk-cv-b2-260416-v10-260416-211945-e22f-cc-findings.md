## Issues Found

1. **[CRITICAL] TDZ risk — `tableIsClosed` useMemo placed BEFORE `isCancelledOrder` helper.**
   Fix 3 Step 3.2 instructs: *"Точное место вставки: НЕПОСРЕДСТВЕННО ПЕРЕД `const ordersSum = React.useMemo` (строка ~490)"*. Fix 1 Step 1.1 places `isCancelledOrder` *"сразу ПОСЛЕ `tableOrdersTotal` useMemo (строка 514-523) и ДО `submittedTableTotal` (строка 525)"*, i.e. at line ~524. After Fix 1 is applied, `tableIsClosed` (line ~490) sits **above** `isCancelledOrder` (line ~524). But `tableIsClosed` callback references `isCancelledOrder` — on first render React invokes `useMemo`'s callback synchronously during render, the const `isCancelledOrder` is still in TDZ ⇒ `ReferenceError` crash on mount.
   **PROMPT FIX:** Either (a) move Fix 3 Step 3.2 insertion location to **AFTER** the new `renderedTableGuestCount` useMemo (i.e. right after the Fix 1 block at ~line 563), or (b) inline the cancelled check in `tableIsClosed` without referencing the helper (e.g. `const raw = (o.status || '').toLowerCase(); const si = getOrderStatus(o); const isCancelled = si?.internal_code === 'cancel' || (!si?.internal_code && raw === 'cancelled'); return !isCancelled;` inside the filter). Option (a) is simpler and preserves DRY.
