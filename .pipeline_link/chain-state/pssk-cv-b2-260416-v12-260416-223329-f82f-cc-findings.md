1. **[CRITICAL] TDZ ReferenceError in Fix 2 Step 2.1 — `statusBuckets` references `isCancelledOrder` declared after it**
   The v12 changelog explicitly states: *"Fix 2 `statusBuckets` — `isCancelledOrder(o)` helper вместо weak inline predicate"*. This modifies `statusBuckets` useMemo at **line 456** (pre-Fix-1, unchanged location post-Fix-1) to call `isCancelledOrder(o)`. But Fix 1 Step 1.1 places the `isCancelledOrder` helper between lines 523–525 → **line 524 post-Fix-1**. Since `React.useMemo(cb, deps)` invokes `cb` synchronously during the render phase, the callback at line 456 runs **before** line 524 has executed → `isCancelledOrder` is in the **Temporal Dead Zone** → `ReferenceError: Cannot access 'isCancelledOrder' before initialization` on **first render**. This is ironic because v11 explicitly addressed the same class of bug for Fix 3 (`tableIsClosed` moved after Fix 1 block), but v12 reintroduced it for Fix 2.

   Inline comment in the prompt (`// defined in Fix 1 above, in scope after Fix 1 applied`) is misleading — in the file, `isCancelledOrder` is declared **below** `statusBuckets`, not above.

   **PROMPT FIX:** Either (a) move the `isCancelledOrder` helper declaration to **before** `statusBuckets` useMemo (insert it at ~line 455 instead of ~line 524; update Step 1.1 placement), **or** (b) keep `statusBuckets` inline predicate (don't call `isCancelledOrder` from it). Option (a) is cleaner because Fix 4 `selfTotal` also uses `isCancelledOrder` from a location after it — a single top-of-hook declaration avoids all ordering concerns. Update line-number estimates for Step 3.2 and Fix 4 Step 4.1 accordingly.
