## Issues Found
1. [CRITICAL] Zero-hit grep checks are shell-unsafe — Several verification steps expect `0 hits` from raw `grep` (`submittedTableTotal`, `ask_bill`, `stale_pending`, `currentTable.status`, `groupLabels`; see prompt lines 486-489 and 1370-1388). In bash, `grep` exits with status 1 when nothing matches. A correct implementation can therefore look like a failed command and derail the run or force the executor to guess whether failure is expected. PROMPT FIX: wrap negative checks in explicit pass/fail shells (`if grep ...; then echo FAIL; exit 1; else echo OK; fi`) or append `|| true` and state that exit code 1 is success for these checks.
2. [MEDIUM] Fix 3 asks for a restaurant-scoped storage key without a verified restaurant identifier in the source — Step 3.0 expects `partnerId`, `restaurantId`, or `currentTable?.partner_id` (prompt lines 885-904), but the actual component only exposes `partner` in the prop list (`CartView.jsx` lines 17-22); there are no `partnerId` / `restaurantId` / `partner_id` hits, and `currentTable` is only used for `name`/`code` (`CartView.jsx` line 385). The fallback grep `partner|restaurant` is too broad and does not tell the executor which scalar field is stable. PROMPT FIX: specify the exact verified expression to use, or instruct the executor to stop if no scalar restaurant identifier exists in this file.
3. [MEDIUM] Working-directory validation is overfit to one machine layout — The prompt says `pwd` should look like `C:/Dev/worktrees/task-*/` and explicitly says `Menu AI Cowork/` is wrong (prompt line 198). That is not a repo invariant; it is a host-specific path assumption. An executor in the correct repo root on a different workstation can be made to think setup is broken. PROMPT FIX: validate only repo-root facts, e.g. current directory contains `pages/PublicMenu/CartView.jsx` and ends with `/menuapp-code-review`.
4. [MEDIUM] The final report template mixes writer and reviewer responsibilities — The prompt’s `Review Instructions` / `Final Rating Table` ask the executor to populate both `CC Rating` and `Codex Rating` columns (prompt lines 1304-1319). In a single implementation run, the second reviewer’s rating does not exist yet, so the executor must guess, leave placeholders, or diverge from the template. PROMPT FIX: split the writer self-check from the cross-review artifact, or mark external-review columns as "filled later".
