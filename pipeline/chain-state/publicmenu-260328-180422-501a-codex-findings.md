# Codex Writer Findings — PublicMenu Chain: publicmenu-260328-180422-501a

## Findings
1. [P1] Fix 1 — `CartView.jsx` still uses a binary `servedOrders` / `activeOrders` split instead of status buckets. `CartView.jsx:92-93` keeps only `servedExpanded` and `activeExpanded`, `CartView.jsx:366-374` groups orders only into served vs non-served, and `CartView.jsx:772-847` still renders `Выдано` / `Заказано`. This cannot produce the required `Подано / Готов / Готовится / Принят / Отправлено` buckets, per-bucket subtotals, or preserved expand state per status. FIX: replace the split with status-keyed buckets (`served`, `ready`, `in_progress`, `accepted`, `new_order`), render only non-empty buckets in the required order, and store UI state in `expandedStatuses`.
2. [P1] Fix 2 — Footer and totals are still wired to the old layout. `CartView.jsx:549-579` shows no `Заказов: X ₸` in the drawer header, `CartView.jsx:980-992` still renders the removed `ИТОГО за визит` card, `CartView.jsx:995` still inserts the old spacer, and `CartView.jsx:1008-1042` renders the sticky footer only when `cart.length > 0`. That misses the required always-visible sticky footer for order-history states and leaves cart/order totals in the wrong places. FIX: move submitted-orders total into the header, put the cart subtotal into the `Новый заказ` section header, remove the visit-total card and spacer, and always render one sticky footer block when `cart.length > 0 || todayMyOrders.length > 0`, switching to outline `Заказать ещё` when the cart is empty.
3. [P1] Fix 3 — Rating is still exposed outside the served state. In `CartView.jsx:457-530`, `renderOrderItems` has no status gate, and `CartView.jsx:489-516` renders `<Rating>` for any order whenever `reviewsEnabled` is true. `CartView.jsx:833-843` also leaves a review CTA inside the active-orders section, while `CartView.jsx:772-802` has no `Подано (N) • Оценить` header chip at all. FIX: pass `showRating` only for the served bucket, remove rating UI from non-served buckets, and move the entry point to a served-bucket header chip that expands `Подано`.
4. [P1] Fix 4 — Status buckets are not collapsed into per-order summary rows. `CartView.jsx:457-529` immediately renders full item lists for every order, `CartView.jsx:474-520` has no per-order collapsed/expanded state, and `CartView.jsx:485` always prints `× {item.quantity}`, including `×1`. This misses the required `HH:MM + first 2 dishes + +N + total + chevron` summary row behavior and the qty rule from CV-23. FIX: add `expandedOrders`, render collapsed summary rows for served/accepted/in_progress/ready buckets, omit `×1`, and keep only the `Новый заказ` section permanently expanded with steppers.
5. [P2] Fix 5 — Guest-facing status labels and the guest-name affordance are still on the old copy. `CartView.jsx:250-278` maps `new` to `Заказано`, keeps `ready` as `Готово`, and has no served/completed fallback to `Подано`; `CartView.jsx:470-472` still shows the per-order badge using that old mapping. In the header, `CartView.jsx:571-577` keeps the editable-name trigger as a plain text button with a pencil icon instead of `Вы: Имя >`. FIX: remap statuses to `Отправлено / Принят / Готовится / Готов / Подано` and replace the pencil affordance with a 44px-tappable name row that shows `Вы: {guestDisplay} >`.
6. [P1] Fix 6 — The order filter still resets at midnight, not at the 06:00 business-day boundary. `CartView.jsx:350-358` compares `new Date(d).toDateString()` to today's date, so orders from the previous business night can leak into the drawer or disappear too early after midnight. FIX: compute `today6am` and filter on `new Date(created_at || created_date || createdAt) >= today6am`.
7. [P2] Fix 7 — The success screen still formats the raw floating-point total. `x.jsx:726-731` renders `{formatPrice(totalAmount)}`, which is exactly the path that can show values like `31.00999999999998 ₸`. FIX: wrap the value before formatting: `formatPrice(parseFloat(Number(totalAmount).toFixed(2)))`.
8. [P1] Fix 8 — Guest name persistence has save/load, but no table-change reset. `x.jsx:1358-1359` restores `menuapp_guest_name`, and `x.jsx:3162` saves it back, but there is no effect tied to the derived table identity after `x.jsx:1560-1634` computes the current table and no cleanup around `x.jsx:2452-2456`, where `currentTableId` changes are already observed for other state. That allows a guest name from one table to survive into another table's session. FIX: add an effect keyed to table identity (`tableCodeParam`/`currentTableId` or equivalent) that calls `localStorage.removeItem('menuapp_guest_name')` and `setGuestNameInput('')`.
9. [P1] Fix 9 — The D3 served-only empty state is not implemented. When the cart is empty and orders exist, `CartView.jsx:772-847` still falls through to the old served/active sections, and `CartView.jsx:850-861` shows the generic `Добавить к заказу` button instead of the dedicated `Ничего не ждёте` screen. There is no `isV8` check above the main bucket rendering, and the footer path for empty-cart history states is non-sticky because `CartView.jsx:1008-1042` only mounts the sticky block for non-empty carts. FIX: compute the served-only state from the new buckets, render the D3 banner above the collapsed `Подано` bucket, and reuse the sticky outline `Заказать ещё` footer for that state.
