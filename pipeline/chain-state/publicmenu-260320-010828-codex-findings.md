# Codex Writer Findings — PublicMenu Chain: publicmenu-260320-010828

## Findings
1. [P1] Loyalty checkout block is gated by the wrong flag — `x.jsx` computes `showLoyaltySection` at line 1937 and passes it into `CartView`, but `CartView.jsx` renders the entire checkout loyalty/discount section behind `showLoginPromptAfterRating` at line 861 instead. That flag only becomes true after the guest starts rating dishes, so fresh carts never get email entry, balance display, or point redemption. FIX: render the checkout loyalty section from `showLoyaltySection`; keep `showLoginPromptAfterRating` only for the post-rating review/login nudge.
2. [P1] Failed instant rating saves permanently lock the stars — `CartView.jsx` immediately calls `updateDraftRating(itemId, val)` at line 710 and then makes the control readonly whenever `draftRating > 0` at line 722. In `x.jsx`, `handleRateDish` catches save failures at lines 2039-2042 but never clears that draft value, so one failed request leaves the item stuck as already rated and the guest cannot retry. FIX: clear the draft rating in the save failure path, or only lock the control from confirmed `reviewedItems`.
3. [P1] Table-code auto-verify cannot retry the same code after a failure — `CartView.jsx` deduplicates submits with `lastSentVerifyCodeRef` at lines 175 and 185, and sets it before firing `verifyTableCode` at line 187. Nothing clears that ref when verification fails, when the cooldown expires, or when the input becomes incomplete again, so a transient error forces the guest to edit the digits just to retry. FIX: reset `lastSentVerifyCodeRef` on verification failure, on cooldown unlock, and when the entered code is shorter than the required length.
4. [P1] Review reward banner appears before any dish is actually reviewable — `CartView.jsx` sets `shouldShowReviewRewardHint` from `isReviewRewardActive && (myOrders?.length > 0)` at line 387. That shows the `+N for review` prompt as soon as any order exists, even if nothing is ready/served yet and `reviewableItems` is empty. FIX: gate the banner on reviewable/ready items, e.g. ready or served statuses plus `reviewableItems.length > 0`.
