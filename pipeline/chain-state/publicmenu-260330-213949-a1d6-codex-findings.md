# Codex Writer Findings — PublicMenu Chain: publicmenu-260330-213949-a1d6
## Findings
1. [P1] HD-01 pending card state is not implemented — `pages/PublicMenu/x.jsx:1647-1651` still uses the old `helpQuickSent` / `sendingCardId` flow instead of shared `requestStates` + `HELP_COOLDOWN_SECONDS`, and success replaces the whole grid with a generic green confirmation that auto-closes after 2 seconds (`pages/PublicMenu/x.jsx:1693-1696`, `pages/PublicMenu/x.jsx:3755-3760`). The quick-action cards never stay visible in per-card `sending` -> `pending` state with "Запрос отправлен", soft brand tint, and per-card disable behavior. FIX: add shared `requestStates` state with `sentAt`, render stateful card UI for each help type, and remove the full-screen success swap/auto-close path.
2. [P1] HD-02 per-type cooldown and repeat state are missing — there is no `HELP_COOLDOWN_SECONDS`, `requestStates`, or "Напомнить персоналу" logic anywhere in `x.jsx`, and the current buttons are blocked only by global conditions `!!sendingCardId || hasActiveRequest` (`pages/PublicMenu/x.jsx:3748-3753`, `pages/PublicMenu/x.jsx:3770-3774`). That prevents independent waiter/bill/napkins/menu cooldowns and cannot transition a single card into `repeat` after 90/150/240 seconds. FIX: derive each card state from `requestStates[type].sentAt` plus `HELP_COOLDOWN_SECONDS[type]`, then render `repeat` with the required reminder copy and resend only that type.
3. [P1] HD-03 relative timer is absent on pending cards — the help drawer code stores no per-request timestamp and contains no help-specific `getRelativeTime()`, 30-second timer refresh, or `visibilitychange` recalculation. The only help success text is static (`pages/PublicMenu/x.jsx:3755-3760`), so cards cannot show "Только что" / "N мин назад" and keep that value fresh. FIX: store `sentAt` in `requestStates`, add `getRelativeTime(sentAtMs)`, and refresh the computed labels every 30 seconds plus on tab visibility changes.
4. [P1] HD-04 "Другое" is still the old minimal textarea flow — tapping the card only toggles `showOtherForm` and selects preset `other` (`pages/PublicMenu/x.jsx:3785-3788`), then renders a textarea bound to shared `helpComment` (`pages/PublicMenu/x.jsx:3795-3804`) with one sticky submit button (`pages/PublicMenu/x.jsx:3813-3824`). The required chips, `otherExpanded` / `otherDraft` state, 100-char counter, cancel action, whitespace-only guard in the inline form, and truncated pending preview are all missing. FIX: replace this branch with the HD-04 inline form structure and store the submitted preview inside `requestStates.other`.
5. [P2] HD-05 request state is not persisted and is reset on every reopen — `openHelpDrawer` and `closeHelpDrawer` explicitly clear the help drawer UI state back to idle (`pages/PublicMenu/x.jsx:1661-1665`, `pages/PublicMenu/x.jsx:1672-1676`), and there is no help-specific `localStorage` key such as `helpdrawer_${tableId}` anywhere in the file. Closing and reopening the drawer therefore loses all help request status instead of restoring entries newer than 240 seconds. FIX: persist only `requestStates` to a table-scoped key, restore/filter it on mount, and clear it on table change or server-send failure.
6. [P2] HD-06 undo toast is not implemented; requests are submitted immediately — `handleQuickSend()` only marks the card as sending and selects the preset (`pages/PublicMenu/x.jsx:1679-1684`), and the follow-up effect calls `submitHelpRequest()` immediately once the preset lands (`pages/PublicMenu/x.jsx:1686-1705`). There is no `undoToast` state, no 5-second countdown, and no path that returns the card to idle before the server send. FIX: add `undoToast: { type, expiresAt } | null`, delay confirmation for 5 seconds, render the in-drawer undo toast, and cancel back to idle if the user taps "Отменить" in time.
7. [P2] HD-07 active-request badge is not wired to the external Help button — the trigger only renders `<HelpFab fabSuccess={fabSuccess} isSendingHelp={isSendingHelp} isHelpModalOpen={isHelpModalOpen} onOpen={openHelpDrawer} t={t} />` (`pages/PublicMenu/x.jsx:3714-3722`), and `x.jsx` has no `activeRequestCount` derivation at all. That means the bell/help trigger cannot show the required pending-count badge when requests remain active. FIX: lift shared request state high enough to compute `activeRequestCount` from pending entries and pass that count into `HelpFab` for badge rendering.
8. [P2] HD-08 active-requests summary block is missing — the drawer header area shows only the table row and a generic one-line `hasActiveRequest` banner (`pages/PublicMenu/x.jsx:3742-3753`). There is no `pendingRequests` list, no total count, and no per-request relative time block above the 2x2 grid, so reopening the drawer cannot show "Активные запросы: N" with itemized rows. FIX: derive `pendingRequests` from `requestStates`, render the summary block above the grid, and reuse `getRelativeTime()` for each summary row.
