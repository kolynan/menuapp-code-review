# Codex Writer Findings - PublicMenu
Chain: publicmenu-260320-173330

## Findings
1. [P0] Order can be committed before its items exist - `x.jsx` creates `Order` and even redeems loyalty points before `OrderItem.bulkCreate` succeeds (`x.jsx:2434-2468`, `x.jsx:2810-2841`). If item creation fails, the guest sees a retry error but staff gets a partial/empty order and redeemed points can already be gone. FIX: create `OrderItem`s immediately after `Order.create`, move loyalty side effects after successful item creation, and best-effort cancel/delete the order if item creation fails.
2. [P0] Session polling re-arms after unmount - `useTableSession.jsx:671-685` schedules the next timeout without checking `cancelled`. A poll that finishes after cleanup can start a new timeout chain and keep updating stale state. FIX: guard `scheduleNext()` and the timeout callback with `if (cancelled) return` before scheduling.
3. [P1] Cart survives mode switches with invalid dishes - cart storage is keyed only by `partnerId` (`x.jsx:172-184`, `x.jsx:1405-1407`) and `handleModeChange` only flips `orderMode` (`x.jsx:2203-2223`). A hall-only dish can stay in cart after switching to pickup/delivery and be submitted through the wrong channel. FIX: key cart storage by partner plus mode, and filter or clear incompatible items on mode change and restore.
4. [P1] Pickup and delivery checkout drops loyalty and discount UI - `x.jsx` passes loyalty props into `CheckoutView` (`x.jsx:3181-3195`), but `CheckoutView.jsx:7-27` does not accept or render them. Guests cannot enter email, redeem points, or see the loyalty block in non-hall checkout. FIX: add the missing props and render the same loyalty and redeem section here, or extract a shared checkout block.
5. [P1] Hall confirmation shows the wrong total - hall flow snapshots `confirmedTotal` with `cart.reduce(...)` (`x.jsx:2539-2556`) while pickup and delivery use `finalTotal` (`x.jsx:2876-2892`). Discounts and redeemed points are ignored on the hall confirmation screen. FIX: use `finalTotal` for hall confirmation too, and keep the savings breakdown separate.
6. [P1] Semantic palette migration is incomplete - hardcoded indigo, green, and cold-white styles remain in `MenuView.jsx:85,104,162,203,229,240,252`, `CheckoutView.jsx:37,76,165-173`, `ModeTabs.jsx:21,33`, `CartView.jsx:469,506,940,1254`, and `x.jsx:759,2983`. The batch goal in the task context will still ship visibly old colors. FIX: replace these hardcoded utility classes with the terracotta semantic palette, including hover and disabled states.
7. [P2] Verified-table block still regresses the cart drawer on mobile - after verification, `CartView.jsx:1066-1082` still renders a second "table verified" block even though the header already shows the table, and the info trigger is still a tiny icon-only target. This wastes vertical space and pushes the CTA lower on small screens. FIX: hide the redundant verified block after success and replace the tiny info trigger with a 44px labeled button.
8. [P2] Scroll reset after table verification targets the wrong element - `CartView.jsx:148-149` queries `[data-radix-scroll-area-viewport], [role="dialog"]`, but the real scroll container is the overflow wrapper in `x.jsx:3258`. On long carts the drawer can stay scrolled near the bottom after verification. FIX: pass a ref to the actual overflow container and scroll that ref instead of querying the document.
9. [P2] Cart info modal backdrop is not touch-friendly - the modal closes with `onMouseDown` only (`CartView.jsx:1183-1187`). On phones, tapping outside the sheet is unreliable because touch interactions do not map cleanly to mouse-down handlers. FIX: use `onClick` or `onPointerDown` backdrop handling, or the dialog component's built-in outside-click behavior.
10. [P2] Remaining i18n leaks are still user-visible - bill-request failures use `toast.error(t('toast.error'))` with no fallback (`x.jsx:2959`), and the reward-email mini-form hardcodes `"email@example.com"` and `'...'` (`CartView.jsx:522,547`). These can surface raw keys or untranslated English text. FIX: add dedicated translation keys and route all user-facing copy through `t` or `tr`.
