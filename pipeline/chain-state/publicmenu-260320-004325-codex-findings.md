# Codex Writer Findings — PublicMenu
Chain: publicmenu-260320-004325

## Findings
1. [P1] Loyalty panel is gated by the wrong flag — `CartView.jsx:859-980` renders the checkout loyalty/discount UI only when `showLoginPromptAfterRating` is true, so a fresh cart cannot enter email, see balance, or redeem points during normal checkout. FIX: gate checkout loyalty UI with a dedicated checkout flag such as `showLoyaltySection`; keep `showLoginPromptAfterRating` only for the post-rating nudge.
2. [P1] Failed rating saves leave dishes permanently locked — `CartView.jsx:705-725` makes the stars readonly as soon as `draftRating > 0`, but failed async saves never clear that draft state, so the guest cannot retry. FIX: clear the draft rating on save failure, or only lock from confirmed `reviewedItems`.
3. [P1] Same table code cannot be retried after a failed auto-verify — `CartView.jsx:174-190` dedupes on `lastSentVerifyCodeRef.current` and never clears it on error, unlock, or partial re-edit, so the same code stops auto-submitting after one failure. FIX: reset `lastSentVerifyCodeRef` when verification fails, cooldown ends, or the input becomes incomplete.
4. [P1] Review reward hint appears before anything is reviewable — `CartView.jsx:386` uses `myOrders?.length > 0`, so the "За отзыв +N" banner shows even when no order is ready/served yet. FIX: gate the banner on `reviewableItems.length > 0` and the relevant ready/served statuses.
5. [P1] Cart can be closed while submit is in flight — `CartView.jsx:464-466` still runs `onClose()` or `setView("menu")` during `isSubmitting`, so the guest can dismiss the drawer mid-request and lose the visible submit result. FIX: disable the close control and short-circuit every close path while `isSubmitting` is true.
6. [P2] Order status badges lost accepted/icon differentiation — `CartView.jsx:240-252`, `584-600`, and `682-684` do not map `accepted` and render only `status.label`, so sent/accepted/cooking/ready states collapse into near-identical text. FIX: add an `accepted` fallback and render `{status.icon}` with the label everywhere status is shown.
7. [P2] Drawer scroll is not reset after table verification — `CartView.jsx:136-140` only clears attempt counters when verification succeeds; it no longer restores the drawer scroll position. FIX: detect the `false -> true` verification transition and scroll the nearest drawer container back to the top.
8. [P2] Guest code leaks into the drawer header — `CartView.jsx:274-283` appends `#${effectiveGuestCode}` to the main guest label even when `hallGuestCodeEnabled` is off, exposing an internal waiter code in general UI. FIX: keep the code only in the dedicated waiter-code block.
9. [P2] Verified-table state wastes mobile space and keeps tiny info targets — `CartView.jsx:1007-1060` and `1046-1049` keep the large yellow verification card after success and use small icon-only info buttons that are below mobile touch-target guidance. FIX: collapse the block after success and replace the info controls with 44px accessible buttons or remove them once verified.
10. [P2] Loyalty amounts bypass app locale and currency formatting — `CartView.jsx:398-399`, `922-925`, `958-961`, `989`, and `1033` hardcode `toLocaleString('ru-RU')`, `₸`, and manual point text instead of the page formatter. FIX: use shared locale-aware formatters and avoid hardcoded locale/currency output.
11. [P2] Reward email flow shows success without validation or confirmed persistence — `CartView.jsx:514-524` accepts any non-empty string, immediately toasts success, and hides the form before any verified save/lookup result exists. FIX: validate the email format, await the real persistence/load outcome, and show an error state on failure.
12. [P2] Submit error copy can falsely claim the order was saved — `CartView.jsx:1216-1217` always says "Ваш заказ сохранён. Попробуйте снова" even when submit may have failed outright. FIX: switch to neutral retry copy, or branch on a confirmed persisted state before saying the order was saved.
13. [P2] Eight-digit table codes overflow narrow phones — `CartView.jsx:100-101` allows up to 8 digits, but `1074-1083` renders fixed `w-9` slots with `gap-2`, which exceeds a 320px viewport. FIX: make slot size/gap responsive or fall back to a single-input layout for longer codes.
14. [P2] `loyalty_redeem_rate = 0` renders the wrong monetary value — `CartView.jsx:925` uses `(partner?.loyalty_redeem_rate || 1)`, so an explicit zero rate is replaced by `1`. FIX: use nullish coalescing and format the computed amount through the normal price formatter.
15. [P2] Other-guests review CTA invents a `+10` bonus fallback — `CartView.jsx:623` multiplies by `(partner?.loyalty_review_points || 10)`, so partners with no review reward configured still advertise bonus points. FIX: reuse `reviewRewardPoints`, show the bonus text only when it is positive, and otherwise render a neutral CTA.
16. [P3] Several icon-only controls are missing accessible names — `CartView.jsx:421-427`, `449-450`, `464-468`, `1007-1014`, and `1056-1063` rely on glyphs or `title` text only, so screen readers get poor labels and some controls are effectively unnamed. FIX: add `aria-label` to every icon-only button and keep hit targets large enough for touch.
17. [P3] Reward-email success timer is not cleaned up on unmount — `CartView.jsx:521-524` schedules delayed state updates with `setTimeout` and no cleanup, so closing the drawer can still fire state setters after unmount. FIX: store the timeout id in a ref and clear it in `useEffect` cleanup, or remove the timer-driven reset.
