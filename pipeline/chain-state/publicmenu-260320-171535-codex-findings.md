# Codex Writer Findings — PublicMenu
Chain: publicmenu-260320-171535

## Findings
1. [P0] Polling timer leak survives unmount in `useTableSession` — `scheduleNext()` in [useTableSession.jsx](/C:/Dev/menuapp-code-review/pages/PublicMenu/base/useTableSession.jsx#L670) always registers the next `setTimeout` even after cleanup has set `cancelled=true`. If `pollSessionData()` is still in flight during unmount, the callback at lines 671-674 can create a fresh orphan timer and continue polling a stale component. FIX: guard `scheduleNext()` with `if (cancelled) return` before `setTimeout`, and reschedule only after the current poll finishes and confirms the effect is still active.
2. [P1] Pickup/delivery checkout regressed back to fullscreen instead of drawer — [x.jsx](/C:/Dev/menuapp-code-review/pages/PublicMenu/base/x.jsx#L2933) still sends guests to `view === "checkout"` via `setView("checkout")`, and the page renders [CheckoutView](/C:/Dev/menuapp-code-review/pages/PublicMenu/base/x.jsx#L3171) as a separate screen while the `Drawer` only supports `drawerMode === 'cart'` at [x.jsx](/C:/Dev/menuapp-code-review/pages/PublicMenu/base/x.jsx#L3249). This reintroduces the mobile UX regression documented in `BUGS.md`: pickup/delivery breaks the bottom-drawer flow and loses the consistent close/submit behavior. FIX: route pickup/delivery checkout through drawer state as well (for example `drawerMode: 'cart' | 'checkout'`) and render checkout inside `DrawerContent` instead of switching the whole page to `view === "checkout"`.
3. [P2] Verified-table block still duplicates the header after successful verification — [CartView.jsx](/C:/Dev/menuapp-code-review/pages/PublicMenu/base/CartView.jsx#L1021) always renders the amber “online order” card, and when `isTableVerified === true` it still shows a second “Стол подтверждён” status inside that block at [CartView.jsx](/C:/Dev/menuapp-code-review/pages/PublicMenu/base/CartView.jsx#L1065). On mobile this repeats information already shown in the drawer header and keeps dead UI visible even when there are no bonus/discount details to show. FIX: restore a `shouldShowOnlineOrderBlock`/`hasOnlineBenefits` gate so the verification card disappears once the table is verified unless there is actual savings/loyalty content to display.
4. [P2] Partner mobile grid setting is ignored on phones — [MenuView.jsx](/C:/Dev/menuapp-code-review/pages/PublicMenu/base/MenuView.jsx#L43) computes `menu_grid_mobile`, but render hardcodes `grid-cols-2` at [MenuView.jsx](/C:/Dev/menuapp-code-review/pages/PublicMenu/base/MenuView.jsx#L276). Any partner configuration for 1-column mobile layout is silently dropped. FIX: use `MOBILE_GRID[mobileCols]`/`gridColsClass` in the mobile branch instead of hardcoding `grid-cols-2`.
5. [P2] Several mobile icon controls are still too small or unlabeled — list-mode quantity controls in [MenuView.jsx](/C:/Dev/menuapp-code-review/pages/PublicMenu/base/MenuView.jsx#L102) and [CheckoutView.jsx](/C:/Dev/menuapp-code-review/pages/PublicMenu/base/CheckoutView.jsx#L58) are below the 44px touch target guideline and have no localized `aria-label`; the cart close button at [CartView.jsx](/C:/Dev/menuapp-code-review/pages/PublicMenu/base/CartView.jsx#L478) is icon-only with no `aria-label`; the table-code info button at [CartView.jsx](/C:/Dev/menuapp-code-review/pages/PublicMenu/base/CartView.jsx#L1076) is also icon-only and smaller than 44px. This is a real mobile usability/accessibility regression in a QR-menu flow. FIX: make every icon-only control at least `44x44` and add translated `aria-label`s for add/remove/close/info actions.
6. [P2] Submit-error fallback text falsely claims the order was saved — the cart error state at [CartView.jsx](/C:/Dev/menuapp-code-review/pages/PublicMenu/base/CartView.jsx#L1233) still falls back to `Ваш заказ сохранён. Попробуйте снова`, which is the same misleading copy previously tracked in `BUGS.md` as PM-038. If the locale key is missing or stale, the UI tells guests the order exists when it may not. FIX: replace that fallback with neutral retry text matching the safer copy already defined in [x.jsx](/C:/Dev/menuapp-code-review/pages/PublicMenu/base/x.jsx#L407).
7. [P3] Reward-email microcopy still bypasses i18n — the reward-email form in [CartView.jsx](/C:/Dev/menuapp-code-review/pages/PublicMenu/base/CartView.jsx#L518) hardcodes `"email@example.com"` and `'...'` instead of using translation keys. This leaks English-only copy into localized menus. FIX: move the placeholder and saving/loading text into `t()/tr()` keys, for example `form.email_placeholder` and `common.saving`.
