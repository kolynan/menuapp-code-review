## TG Findings Streaming Review

### Summary
Implemented per-worker JSONL findings streaming for V7 code-review tasks across the common helpers, supervisor, Claude writer launcher, and Codex reviewer launcher.

### Changes
- Added `Get-V7FindingStreamPath`, `Add-V7FindingStreamEntry`, `Read-V7FindingStreamEntries`, and `Sync-V7ReviewFindingsToStream` to `pipeline/v7/scripts/V7.Common.ps1`.
- Extended `Ensure-V7TelegramState`, added `Update-V7TelegramFindingStreamState`, rendered a `--- Findings ---` section in `New-V7WorkflowTelegramMessageText`, and wired stream refresh into both the 15-second loop and 2-minute heartbeat in `pipeline/v7/scripts/Start-TaskSupervisor.ps1`.
- Added live findings stream setup and prompt instructions to `pipeline/v7/scripts/Invoke-ClaudeWriter.ps1`, plus a final commit milestone backfill entry after successful completion.
- Added live findings stream setup and prompt instructions to `pipeline/v7/scripts/Invoke-CodexReviewer.ps1`, plus final reviewer JSON backfill through `Sync-V7ReviewFindingsToStream`.
- Fixed the stream reader so a trailing newline does not advance the cursor past future entries, and guarded reviewer backfill so empty payloads do not emit blank findings.
